# v0.3.15 Cafe Scene Backdrop

Final surface of the illustrated art standard (issue #3): the Bear Cafe
environment, as a softened backdrop on the child home screen. Direction approved
by Juan from the concept render.

## Before Code

### Root Cause

The child home screen is stacked cards on a flat gradient; issue #3 §5 calls for
a cafe environment (counter, window, sign) to set the mood.

### Correct Fix Must Touch

- `src/modules/home/cafe-scene.ts` (new) — the approved cafe scene as a local
  inline-SVG string + `createCafeBackdrop()` returning a backdrop element.
- `src/modules/home/HomeScreen.ts` — inject the backdrop as the first child of
  the home container.
- `src/styles/child-ui.css` — `#home-screen` becomes a positioned, isolated
  wall-colored surface; `.cafe-backdrop` sits behind the content (`z-index: -1`),
  softened, with the scene anchored full-width to the bottom.
- `package.json` / `package-lock.json` — version bump to `0.3.15`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- **Safety boundary:** inline SVG only (no external assets/network/deps); no
  `http(s)` beyond the standard SVG xmlns (as with the bear/food art).
- **Scoped to the home screen only** — the backdrop is injected by HomeScreen, so
  it does not appear behind the individual activities (where a cafe theme would
  clash). `aria-hidden` + `pointer-events: none` keep it decorative.
- No change to activities, evidence model, mastery, transfer, storage, parent
  panel, or the child gate. The four home choices, greeting, and parent access
  are unchanged and readable (the scene is at 0.5 opacity behind them).

## Cold Diff Audit

### Change By Change Reconstruction

- `cafe-scene.ts` (new) — `CAFE_SCENE_SVG` (wall + wainscot, a sunny window with
  a windowsill plant, a hanging "Bear Cafe" sign, a pendant light, and a wood

- version → `0.3.15`; `MVP_BASELINE.md` note.
