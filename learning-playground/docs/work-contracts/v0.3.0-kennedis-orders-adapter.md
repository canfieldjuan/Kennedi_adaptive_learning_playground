# v0.3.0 Kennedi's Orders Adapter Contract

## Before Code

### Root Cause

`Kennedi's Orders` exists on a separate game-designer branch based on `v0.2.2`, while current main is `v0.2.10`. Current main has no Kennedi's Orders module, no registered Bear Cafe activities, no route dispatch for the game, and no game-specific truth guards for its richer transfer contexts. Directly merging the old branch would also import out-of-scope `Nature Camera Safari` work and old app wiring.

### Correct Fix Must Touch

- `docs/work-contracts/v0.3.0-kennedis-orders-adapter.md` to record the before-code contract and final audit.
- `src/modules/kennedis-orders/` for module-local Bear Cafe types and runtime.
- `src/content/activities/kennedis-orders-*.json` for exactly the six Kennedi's Orders activities.
- `src/content/activity-catalog.ts` to register exactly those six activities.
- `src/app/main.ts` to dispatch only `tap_then_place` activities whose content game is `kennedis-orders`.
- `src/styles/child-ui.css` for scoped Bear Cafe presentation and reduced-motion handling.
- `tests/modules/kennedis-orders.test.ts` for schema, safety, events, evaluation, chain, and no-Nature-Camera boundaries.
- `tests/contract/validate-activities.test.ts` only as needed to add game-specific truth guards for Bear Cafe rich contexts.
- Status/version docs and package metadata if verification passes.

### Must Not Change

- Do not import or wire `Nature Camera Safari`.
- Do not expose Cafe on the child home screen in this slice.
- Do not change activity schema, event schema, mastery engine, transfer coverage engine, recommendation engine, storage, parent panel behavior, parent approval semantics, or local export/reset behavior.
- Do not add backend, auth, cloud sync, AI tutor chat, open web content, rewards, streaks, leaderboards, public sharing, or automatic child routing.
- Do not weaken safety, parent approval, transfer truth, or mastery rules to make the game fit.

## Contract Amendments

- The first browser pass showed the Bear Cafe make-stage controls and delivery action could fall below the default viewport. The adapter scope was amended to include layout fixes inside scoped Bear Cafe CSS/runtime only, because a child-facing game adapter is not acceptable if core actions require hidden scrolling on a common viewport.
