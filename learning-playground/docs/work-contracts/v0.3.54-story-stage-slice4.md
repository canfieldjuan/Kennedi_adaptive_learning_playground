# v0.3.54 Kennedi's Story Stage — slice 4: three story families + content pack

The content slice: Lost Friend generalized, The Broken Thing and The
Special Delivery authored (spec §8), and the original character/setting
pack (§9) — so the Pick Three shelves from slice 3 hold real choice.
Spec §25/slice-4: three families, original pack, multiple valid paths,
ending variants, story graph tests.

## Before Code

### Root Cause (slice)

Slice 3 shipped the selection mechanism over a single authored trio:
one character, one setting, one problem, one family. The spec's product
goal — the child assembles a real story from real ingredients — needs
the two remaining families and enough strong entities that selection is
a genuine creative act. The scene art is the structural gap: slice 1–3
scenes hardcode Poppy in the forest, so entity variation requires a
compositional scene system (backdrop per setting × character pose ×
family beat props) before any new content can render coherently.

### Required Change Surface

- `src/modules/story-stage/story-art.ts` — restructured into a
  compositional system (§16 layering):
  - **Character builders**: Poppy (exists) + Finn the friendly dragon,
    Robo the helpful robot, Luna the gentle unicorn, Milo the playful
    puppy — each a `(mood, x, y)` builder with the four established
    moods (happy / worried / curious / celebrating), purple-ink
    illustrated standard, distinct silhouettes.
  - **Setting backdrops**: forest (exists) + moon castle, cozy town,
    cloud village — each a `(extra) => svg` 400×250 stage with ground
    strip and low-contrast decoration.
  - **Scene builders**: per family beat, `(ctx) => svg` where ctx
    carries the selected character + setting art keys —
    `storySceneSvg(artKey, ctx)` replaces the fixed SCENES record.
    Lost Friend beats reuse the existing prop compositions on any
    compatible backdrop; new prop sets for Broken Thing (music box,
    peek-light, winding, tiny toolkit, helping friend) and Special
    Delivery (berry basket, wobbly bridge, butterfly meadow, carry-high,
    wagon).
  - **Card + choice art**: cards for the four new characters, three new
    settings, two new problems (each communicating its idea visually);
    choice art for the four new decision options per new family.
- `src/modules/story-stage/story-pack.types.ts` — `{settingDetail}`
  joins the closed token set; `StorySetting` gains `detail`;
  `StoryProblem`'s friend fields (`friendLabel`/`friendPhrase`/
  `friendThem`) become OPTIONAL (only the Lost Friend problem carries a
  friend).
- `src/modules/story-stage/story-resolver.ts` — token map built from
  available entity fields only (friend tokens present only when the
  problem defines them; unknown-token throw unchanged).
- `src/modules/story-stage/story-validator.ts` — new rule making
  cross-entity coherence structural: **dry-resolve every supported
  (family × character × setting × problem) combination** and report
  each failure. This subsumes per-template token checks against
  optional entity fields and enforces §13's "no branch may accidentally
  switch character, setting, or problem / no missing assets" at the
  data level.
- `src/modules/story-stage/first-tale.ts` — becomes the full
  `FIRST_STORY_PACK`: 5 characters, 4 settings, 3 problems, 3 families.
  - **Lost Friend** (existing 8 scenes; narration templates unchanged
    except intro gains `{settingDetail}` — resolved forest intro stays
    byte-identical — and the family title generalizes to
    "…'s Great Adventure" since the family now supports cozy town).
    Compatibility: poppy/finn/luna/robo × forest/cozy-town ×
    missing-friend (Milo excluded: a puppy hero hunting a puppy friend
    reads as a duplicate).
  - **The Broken Thing** (§8B): quiet music box with a loose wheel →
    examine → HOW decision (wind gently / look inside — first attempt
    changes the situation, never failure) → finishing decision (tiny
    toolkit / ask a friend) → two endings (fixed alone / fixed
    together). All 5 characters × cozy-town/moon-castle/cloud-village.
  - **The Special Delivery** (§8C): sunshine-berry basket for the
    surprise party → route decision (wobbly bridge / butterfly meadow,
    gentle obstacles) → protect decision (carry high / little wagon) →
    two happy-recipient endings. All 5 characters ×
    forest/cloud-village/moon-castle. Distinct from Bear Cafe: no
    orders, no requests, no correctness — one narrated journey.
  - Every scene carries its authored adult cue (beat/keepTrue/ask/
    silly?), safety tone rules hold (§8: no abandonment/danger/blame).
- `StoryStageActivity.ts` — passes the selection's art context into
  `storySceneSvg`; no other flow change.
- `src/styles/child-ui.css` — card sizing for multi-card steps
  (2–3 per row desktop, 2 per row at 390px; modest scrolling allowed
  per §23).
- Tests —
  - `tests/contract/story-pack.test.ts`: full pack validates; every
    supported combination dry-resolves (mutation: a template token the

    pins kept (intro/scenes byte-identical; title pin updated to the

  - `tests/modules/story-stage.test.ts`: graph proofs iterate ALL

## Contract Amendments

- **Six-beat structure delivered as 8-scene graphs.** Each family's
  §8 six-beat structure maps to eight authored scenes (intro, problem,
  decision, 2 consequences, decision, 2 endings) with a shared
  `maxPathLength` of 6 — same shape the Lost Friend shipped with in
  slice 1, applied uniformly.
- **LF intro cue reworded setting-neutral** ("What do you think {hero}
  sees near {setting}?") since the family now supports cozy town —
  cue wording is the slice-5 draft-quality surface, recorded here so
  the change is deliberate, not silent.
