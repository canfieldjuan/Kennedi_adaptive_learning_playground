# v0.3.52 Kennedi's Story Stage — slice 2: authored model + validator

The content system: a typed story-pack model, a pure graph validator, and
the slice-1 tale migrated into authored data. The runtime plays the same
story through the pack; the child experience is unchanged.

## Before Code

### Root Cause (slice)

Slice 1's tale is hardcoded module data with hand-written tests standing
in for validation. Slices 3–5 need what §11–§14 of the spec define: an
authored model with characters/settings/problems as first-class entities,
controlled narration substitution (never mad-libs), compatibility rules,
adult storyteller cues carried by the content (so Together mode is a
runtime feature later, not a content retrofit), and a pure validator that
makes every future pack provably sound before it ships.

### Required Change Surface

- `src/modules/story-stage/story-pack.types.ts` (new) — the authored
  model: `StoryPack` (characters, settings, problems, families),
  `StoryCharacter` (id, label, spoken intro phrase, pronouns, art key),
  `StorySetting`, `StoryProblem`, `StoryFamily` (id, entry scene,
  max path length, compatible character/setting/problem ids, scenes,
  endings), `PackScene` (id, kind, narration TEMPLATE, adult cue
  TEMPLATE — required on every scene NOW, per §5/§14 — art key, choices
  or next, ending id), `StoryChoice`, `StorySelection`
  (character/setting/problem ids). Narration templates use exactly the
  controlled tokens `{hero}`, `{Hero}`, `{friend}`, `{setting}`,
  `{they}`, `{their}` — resolved from authored entity phrases, never
  free composition (§11).
- `src/modules/story-stage/story-resolver.ts` (new, pure) —
  `resolveStory(pack, familyId, selection)`: validates compatibility,
  substitutes tokens, returns the concrete scene list the runtime plays.
  Deterministic: same pack + selection ⇒ same resolved story.
- `src/modules/story-stage/story-validator.ts` (new, pure) —
  `validateStoryPack(pack)` returning string errors (the
  curriculum-graph validator convention). Detects everything §14 lists:
  duplicate ids; missing entry scenes; missing next/choice targets;
  unreachable scenes; dead ends without endings; cycles; invalid
  character/setting/problem/art references; empty narration; missing
  adult cues; missing choice labels/art; decision choice counts ≠ 2;
  identical choice targets (fake choices); external URLs anywhere;
  incompatible family/selection combinations declared but unsupported;
  paths exceeding the family's max scene count; unresolved or unknown
  narration tokens.
- `src/modules/story-stage/first-tale.ts` — REWRITTEN as
  `FIRST_STORY_PACK`: the Lost Friend family with Poppy (hero), Biscuit
  (friend), the enchanted forest (setting), and the missing-friend
  problem as authored entities; scene narration becomes templates with
  tokens; every scene gains its authored adult cue (beat + fact that
  must stay true + a question to ask + optional silly detail — concise).
  The resolved output is IDENTICAL to slice 1's narration strings
  (test-pinned) so the child experience does not move.
- `StoryStageActivity.ts` — consumes `resolveStory(...)` for the fixed
  selection instead of raw scenes. No UI change.
- Tests — `tests/modules/story-stage.test.ts` updated to the pack path
  (same behavioral assertions); new
  `tests/contract/story-pack.test.ts`: the approved pack validates
  cleanly, resolution is deterministic and byte-stable, and EVERY
  validator rule is proven by a mutation test (a broken clone of the
  pack per rule, each caught with its named error).
- `package.json` / `package-lock.json` — `0.3.52`; `MVP_BASELINE.md`.

## Contract Amendments

- **Token set widened during migration.** The contract named `{hero}`,
  `{Hero}`, `{friend}`, `{setting}`, `{they}`, `{their}`. Reproducing
  slice 1's strings byte-identically required `{Their}` (capitalized
  possessive at sentence start), `{friendPhrase}` ("puppy friend
  Biscuit"), and `{friendThem}` ("him"), while `{they}` had no use site
  and was dropped rather than shipped untested. The set is still closed
  and validator-enforced; entities gained the matching authored fields
  (`introName`/`shortName` on characters, `friendPhrase`/`friendThem`
  on problems). Same rule, more honest vocabulary.
- **Slice-1 types file removed.** `story-stage.types.ts` had no
  remaining consumer once the runtime and tests consume the resolved
  pack; deleted instead of leaving a dead module.
