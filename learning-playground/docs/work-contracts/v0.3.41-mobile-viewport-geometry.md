# v0.3.41 Mobile Viewport Geometry

Issue #42: child controls below the fold at responsive breakpoint boundaries.

## Before Code

### Code Read Cold

- Home uses compact cards through the shared mobile query at
  `src/styles/child-ui.css:2118`, but the large-card query begins at any width
  of 941px at `src/styles/child-ui.css:2930` without considering short height.
- Bear Cafe's short-landscape workbench stays vertically stacked until the
  two-column query begins at 704px at `src/styles/child-ui.css:2923`.
- Short-landscape Word Builder tiles shrink to 48px at
  `src/styles/child-ui.css:2755`, and Coloring swatches shrink to 58px at
  `src/styles/child-ui.css:2861`, while `docs/contracts/child-ui.contract.md:9`
  declares an 80px minimum for primary targets.
- `tests/contract/mobile-child-ui.test.ts:1` reads CSS text and checks for
  substrings. It does not render the app, compute geometry, or probe both sides
  of a breakpoint.

### Runtime Reconstruction

Current-main Chromium measurements confirm the issue's primary claims:

- Home at 940x480: document height 480px; all four cards end by 410px.
- Home at 941x480: document height 710px; Math and Art extend to 654px.
- Home at 1024x600: document height 710px under the same large-card rule.
- Bear Cafe active order at 667x375: document height 517px; food choices span
  341-509px, below the viewport.
- Bear Cafe at 703x375 remains stacked at 517px; at 704x375 the workbench
  becomes two columns, document height is 375px, and food choices end by 252px.
- Bear Cafe at 568x320 is stacked at 517px. A two-column probe keeps all food
  choices between 83px and 251px while retaining 80px choice height.
- Word Builder at 568x320 currently has a 351px document and 48x48px letter
  tiles. A scoped two-column probe fits 80x80px tiles in a 320px document.
- Coloring at 568x320 currently fits with roughly 62px swatches. An 80px swatch
  probe still fits in a 320px document.

### Finding Classification

- **Confirmed - existence:** both reported below-fold paths occur on current
  main with the issue's exact viewport sequences.
- **Confirmed - location:** the controlling rules are the current Cafe query at
  `src/styles/child-ui.css:2923` and Home query at
  `src/styles/child-ui.css:2930`; the issue's historical line numbers are stale.
- **Confirmed - cause:** width-first breakpoint selection changes composition
  without accounting for short height. The target-size contradiction and
  CSS-text-only test gap are also current.
- **Confirmed - severity:** P2. A plausible phone-landscape path hides the
  objects required to complete Cafe, and a one-pixel Home width change hides two
  primary navigation choices.
- **Confirmed - fix direction:** height-aware reflow plus computed browser
  geometry reaches the cause. Further target shrinking would only hide the
  symptom and violate the child interaction standard.

### Root Cause

Responsive layout selection is based on width thresholds that do not represent
the available vertical space. Home enters a height-insensitive desktop mode and
Cafe delays its horizontal workbench until 704px, even though the narrower
layout has enough width for two safe columns. Other short-landscape surfaces
make room by shrinking primary play objects below the declared target size.
Because tests inspect stylesheet text instead of browser geometry, these
boundary failures are invisible to the automated suite.

### Correct Fix Must Touch

- `src/styles/child-ui.css`: make short-wide Home explicitly compact, keep the
  large Home composition behind a sufficient-height condition, and allow the
  Cafe two-column workbench from 568px through 940px landscape.
- `src/modules/phonics-match/WordBuilderActivity.ts`: add one stable layout
  modifier class so short-landscape Word Builder can reflow without affecting
  matcher/blending modes.
- `src/styles/child-ui.css`: reflow that Word Builder into visual-object/slot and
  character/tile columns, hide duplicated title/prompt text only in this spoken
  short-landscape mode, and keep letter tiles at least 80x80px.
- `src/styles/child-ui.css`: keep Coloring swatches at least 80px tall in mobile
  and short landscape.
- `docs/contracts/child-ui.contract.md` and
  `docs/contracts/mobile-child-ui.contract.md`: define target categories
  explicitly. Selectable play objects stay at least 80x80px; wide command
  buttons may be shorter but must remain at least 56px tall and 96px wide;
  utility Home/Repeat controls may be 44px minimum.
- Add Playwright browser geometry coverage for Home, active Cafe, wrong Cafe
  feedback, delivery/completion reachability, Word Builder tiles, and Coloring
  swatches. Probe both sides of 703/704 and 940/941 plus the issue's other
  accepted viewports.
- Keep the existing structural contract tests aligned with the authoritative
  rules, and add Playwright scripts/configuration and ignored artifacts.
- Advance package, baseline, and roadmap to v0.3.41.
