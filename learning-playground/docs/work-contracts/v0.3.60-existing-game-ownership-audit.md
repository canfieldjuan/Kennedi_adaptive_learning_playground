# v0.3.60 Existing-Game Ownership Audit Work Contract

## Before Code

### Root Cause

The permanent Ownership Completion Contract now governs every primary game, but
the repository does not yet contain a current, independently reconstructed
compliance audit that tests each existing primary game's actual implementation
against every required ownership boundary. The v0.3.59 principle work contract
contains a planning snapshot, not a dedicated evidence record derived after the
contract became repository law. Without a cited audit, later retrofit slices
could preserve the wrong state, duplicate an existing capability, alter scored
evidence accidentally, or address a generic completion symptom instead of the
specific failed contract boundary.

The defect is therefore missing code-grounded ownership-compliance knowledge,
not missing decoration and not authorization for a cross-game runtime rewrite.

### Correct Fix Must Touch

- `docs/work-contracts/v0.3.60-existing-game-ownership-audit.md` must state the
  audit contract, verification record, and cold diff audit.
- `docs/audits/v0.3.60-existing-game-ownership-audit.md` must independently
  reconstruct the current behavior of every primary game named by the merged
  Ownership Completion Contract:
  - Words / Word Builder;
  - Bear Cafe / Kennedi's Orders;
  - Number Train;
  - Bear Art Studio;
  - Story Stage.
- For each game, the audit must cite current `file:line` evidence for:
  - the child action that changes, creates, places, repairs, arranges, delivers,
    or directs world state;
  - the current completion object, or the absence of one;
  - at least one unscored ownership choice, or the absence of one;
  - exact-choice continuity through payoff;
  - the child-controlled inspection moment;
  - local bounded preservation and child-facing revisit;
  - the separation between expressive state and scored learning evidence;
  - the bounded-flow budget.
- Each required boundary must be classified as **confirmed**, **contradicted**,
  or **could-not-determine**. Confirmation requires a citation; uncertainty must
  name the missing proof instead of being upgraded to compliance.
- Each game must receive an overall current status and the smallest independent
  retrofit recommendation that reaches its root contract gap. Recommendations
  must preserve exact state already produced by the game and must not combine
  multiple game runtimes into one implementation PR.
- The report must end with an ordered one-game-per-PR retrofit sequence and the
  verification needed for each later slice. Ordering is planning evidence only;
  it does not authorize implementation.

  permanent contracts, dependencies, CI, backend, auth, cloud, rewards, streaks,

  when browser proof is required by the Ownership Completion Contract.

  `docs/work-contracts/v0.3.59-ownership-completion-principle.md`: the four
  child-home worlds plus Story Stage. Legacy direct-route Coloring, Puzzle,
  Video Vault, and absent Nature Camera Safari are outside this audit.
- The audit baseline is merge commit
  `95ecfb22973f8f08d5b4d07a506005533333fdae`.
- Existing durable records may be suitable completion-object sources only when
  current code proves they contain enough exact state and already participate
  in export, clear, cap, and malformed-data boundaries.
- A failed or absent browser path is reported as could-not-determine; it does
  not authorize runtime repair in this slice.
- No blocker is known before inspection.
