# v0.3.55 Kennedi's Story Stage — slice 5: Tell It Together

Mode B (spec §5): the adult improvises over the authored structure. A
parent setting (`narrated` | `together`), the Together runtime (adult
cue panel instead of automatic narration, authored-narration fallback),
and mode persistence tests. Spec §5/§6/§19 + §25/slice-5.

## Before Code

### Root Cause (slice)

The cues have lived in the content since slice 2 — every scene carries
beat / keepTrue / ask / silly — but no surface consumes them, and the
runtime can only narrate. §5's second first-class mode needs: a
parent-owned mode setting (child never chooses, §6), a Together runtime
that presents the cue and stays quiet, and the §5 guarantee that the
session is never stranded (an obvious control plays the authored
narration when the storyteller gets stuck).

  'together'`; `src/core/storage.ts` — `story_mode: 'narrated'` in

  - `createStoryModeSection` (gate-phrase/voice section pattern):

    marked, `saveSettings({...settings, story_mode})` + panel

    no per-scene gating anywhere (§6).
  - Story Stage launch card refreshed (it still describes slice 1):
    status shows the ACTIVE mode + "Pick Three story builder"; Entry

    mode visible to the adult before the story starts" at the launch

- `src/app/main.ts` — passes
  `storyMode: settings.story_mode === 'together' ? 'together' : 'narrated'`
  (anything unexpected degrades to narrated) into `renderStoryStage`.
- `src/modules/story-stage/StoryStageActivity.ts` —
  - Options gain optional `storyMode` (default `'narrated'` — narrated
    tests and behavior unchanged).
  - Setup screen gains a small adult-facing mode badge ("Tell Me a
    Story" / "Tell It Together") — §6 visibility before the story
    starts, at the story surface itself.
  - **Together story phase**: scenes do NOT auto-narrate (opening and
    scene speech suppressed; speech.stop() still fires on transitions
    so a fallback playback never overlaps); the caption starts empty;

    header "Storyteller cue" + collapse/reveal toggle, cue lines (beat,

    line"** fallback that speaks the authored narration AND reveals it
    in the caption (never stranded, §5). Collapse state persists across

    parchment) and never carries a story decision (§19); child
    choices/Continue are untouched. Tell It Again in together mode
    restarts silently.
  - Setup selection speech (step asks + spoken labels) stays in BOTH
    modes — §7's spoken labels are a child selection aid, not story
    narration.
- `src/styles/child-ui.css` — mode badge + cue panel styles (readable

### Explicit Non-Scope / Must Not Change (slice 5)

- Narrated mode: byte-identical behavior (default unchanged; no new
  UI in narrated child flow beyond the setup badge).
- No per-session override UI this slice (the §6 "if it fits cleanly"
  clause — the Parent Panel toggle is one tap behind the gate; a
  second override surface adds ambiguity, recorded as a deliberate
  no).
- Cue WORDING stays as authored in slices 2/4 (polish was always
  slice-5-UI-exists-first; a wording pass is now unblocked but belongs
  to slice 7 alongside the art polish, keeping this slice mechanical).
- No story history (slice 6), no home change (slice 7), no event sink,
  pack/resolver/validator/selection modules at 0 lines, protected
  games at 0 lines.

### Assumptions / Blockers

- The child may see the cue panel text (shared screen) — it contains
  only safe, authored, non-interactive adult guidance; the §19
  requirement is that it never becomes a story-decision target, which
  the layout enforces (separate panel, no choice semantics).
- Blockers: none known.
