# v0.3.52 Bear Video Human Narration

Historical working slice: v0.3.49 before current-main integration.

## Before Code

### Code Read Cold

- The observed screencast contained no audio stream, so it could not establish
  playback quality. The deployed `Bear Bakes Bread` WebM did contain valid Opus
  audio at 48 kHz mono, with no obvious peak clipping.
- The poor sound therefore came from the fixed synthetic narration source, not
  a missing stream, browser setting, response activity, or evidence consumer.
- The clip contains three stable visual beats around 0, 3, and 6 seconds. Its
  original speech occupied the same three windows.
- Video completion is exposure only; the separate tap-choice response owns all
  correct or incorrect vocabulary evidence.
- The manifest had a numeric version but completion metadata cited only its id,
  so replacing media without carrying the version would make old and new
  exposure observations indistinguishable.

### Root Cause

The first local clip embedded poor synthetic narration. Because that audio is
inside the WebM, the app's parent-selected browser voice cannot improve it.
Changing volume alone would preserve the unpleasant source, and replacing the
media without versioned event provenance would blur observations across two
materially different narration revisions.

### Correct Fix Must Touch

- Replace only the WebM audio with the three parent-recorded lines: `Bear mixes
  the dough.`, `Bear bakes the bread.`, and `Bear made warm bread.`
- Trim lead-in silence, preserve natural phrase endings, remove low-frequency
  rumble, match line loudness, and place the lines in the existing visual beats.
- Keep the VP9 video stream unchanged and retain 48 kHz mono Opus output near
  -18 LUFS with true peak below -2 dB.
- Increment the existing manifest version and cite it in video completion event
  metadata without changing the shared event schema or evidence meaning.
- Add focused provenance assertions and update the Video Vault contract,
  baseline, roadmap, package version, and this work contract.

### Acceptance Standard

- The final clip contains the same VP9 video packet stream as the prior clip and
  one 48 kHz mono Opus audio stream.
