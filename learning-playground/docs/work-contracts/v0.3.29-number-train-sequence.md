# v0.3.29 Number Train — Missing Station (arc slice 4)

Fourth slice of the Number Train build arc (issue #43): **number sequence**.
The trip gains "Missing Station" rounds — a short number path along the track
with one missing numeral the child fills in.

## Before Code

### Root Cause (slice)

The trip exercises reading quantities (count) and constructing them (load),
but nothing exercises **number order**: there is no sequence representation,
no missing-number round, and no hinting that walks the track one number at a
time. Number order is a distinct pre-math capability the session plan
reserved a mid-trip position for.

### Correct Fix Must Touch

- `number-train.types.ts` — add `MissingStationRound { kind:
  'missing_station'; sequence; missing_index; choices; prompt }` (the full
  consecutive sequence; the display blanks `missing_index`); union grows.
- `round-plan.ts` — composition update: on the standard six-round trip,
  round 4 stays Load-the-Train and **round 5 becomes Missing Station**
  (shorter authored trips keep their single build round; sequences appear
  only on trips of ≥6). Sequence generation: a four-number consecutive
  ascending path anchored near the ramp quantity, clamped to `1..max`;
  seeded missing position; numeral choices reuse the neighbor-distractor
  builder. `validateRound` gains sequence rules: length ≥ 3, strictly
  consecutive ascending, all values `0..max`, `missing_index` in range,
  and the choice rules (answer present exactly once, no duplicates, all
  within bounds).
- `NumberTrainActivity.ts` — `renderMissingStationRound`: the stage renders
  a **number path of station signs along the track** (engine at the front;
  one sign per sequence value; the missing one shows "?" and is emphasized);
  numeral choices below. Correct tap → the blank fills with the numeral,
  normal round-complete path. Wrong tap → `incorrect`; after two wrong → one
  bounded structural hint that **walks the track one number at a time**
  (speaks the surrounding sequence with the blank, asks "what comes after
  N?" — or "before N" when the blank leads —, pulses the signs in order,
  highlights the correct numeral). `correctAnswerOf` covers the new kind.
- `child-ui.css` — number-path/station-sign styles (missing/filled states,
  walk-along pulse) with reduced-motion coverage.
- `tests/modules/number-train.test.ts` — composition across seeds (round 5
  is `missing_station`); the seed sweep validates sequences (consecutive,
  bounded, choices valid); missing-round behavior (renders the path with one
  "?", correct tap fills and advances, two wrong → walk-along hint +
  highlighted numeral); full mixed trip (three round kinds) still emits
  `completed` exactly once; validation rejects non-consecutive sequences and
  out-of-range missing indexes.
- `package.json` / `package-lock.json` — version bump to `0.3.29`.
- Docs note in `MVP_BASELINE.md`.

### Explicit Non-Scope / Must Not Change

- **Evidence stays `counting`** this slice; the `number_sequence` skill and
  the full curriculum split are slice 5 (documented limitation, not hidden).
- Count/Load round behavior, journey/arrival/replay, envelope JSON,
  dispatch, home route, curriculum, `src/core/`, other runtimes — untouched.
- No adaptive routing, reward loops, timers, or unbounded randomness.
