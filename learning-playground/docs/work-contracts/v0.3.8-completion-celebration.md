# v0.3.8 Completion Celebration

Fifth slice of issue #3 (game richness backlog) applied to Kennedi's Orders.

## Before Code

### Root Cause

The complete stage is a bear `popIn` + "Order delivered." text with no payoff
moment (`renderCompleteStage` in `KennedisOrdersActivity.ts`). Issue #3 names
this "Minimal completion payoff — no confetti / sticker / particle burst."

### Correct Fix Must Touch

- `src/modules/kennedis-orders/KennedisOrdersActivity.ts` — render a one-time
  celebration burst on the complete stage.
- `src/styles/child-ui.css` — scoped `.bear-cafe-celebrate*` styles + a
  `cafeCelebrateBurst` keyframe, added to the `prefers-reduced-motion` guard;
  make `.bear-cafe-complete` a positioned container for the overlay.
- `package.json` / `package-lock.json` — version bump to `0.3.8`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change (safety)

- **Not a reward loop.** The safety contract forbids random reward loops and
  streaks. The burst is deterministic (12 pieces at fixed index-derived angles,
  no `Math.random`), identical on every completion, and plays once — it is
  positive *completion feedback*, not a variable/random reward and not a streak.
- No change to the evidence model, activity JSON, schema, mastery, transfer,
  recommendation, storage, parent panel, home grid, or the child gate.
- No external assets/network/deps; emoji + CSS only (no `http(s)` introduced).
- Motion stays preschool-safe: brief (~1.4s), gentle, reduced-motion-guarded
  (the burst is hidden under reduced motion; the bear popIn + text baseline
  remains).
