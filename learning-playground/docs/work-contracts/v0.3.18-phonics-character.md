# v0.3.18 Word game ‚Äî recurring character ("the word comes alive")

Third slice of the Word-game build arc (issue #27). Adds **Pip**, a recurring
illustrated phonics character on the `phonics-match` runtime: Pip mouths the
target sound (child sees how the sound is made) and "comes alive" (cheers) when
the word is found. Affect/reward-feel only ‚Äî no new phonics skill, no reward loop,
no auto-difficulty routing.

## Before Code

### Root Cause

The Word game's intrinsic reward ‚Äî the thing that makes a correct match feel
alive ‚Äî does not exist. The `phonics-match` runtime shows a prompt, a choice
grid, and text feedback; a correct answer just locks the grid and speaks a line.
There is no recurring character, nothing that shows the child *how the sound is
made* (mouth shape), and nothing that makes the found word "come alive". The user
named this as the centerpiece of the Word game.

### Correct Fix Must Touch

- `src/modules/phonics-match/phonics-character-art.ts` (new) ‚Äî Pip's illustrated
  inline SVG (local, no external assets ‚Äî safety boundary), matching the Bear
  Cafe standard (ink `#3a2461`, viewBox 0 0 100 100, a mouth/eye swap per state).
  `mouthForSound(letter)` maps the initial sound to a child-legible viseme
  (bilabial /b//m/, sibilant /s/, open /c/=/k/, tongue /t/, else idle);
  `renderPhonicsCharacterArt(mouth)` returns the SVG; `cheer` is the alive beat.
- `src/modules/phonics-match/PhonicsMatchActivity.ts` ‚Äî insert a character panel
  between the prompt and the grid; set Pip's mouth from `content.target_sound`;
  pulse Pip while the prompt is spoken (initial render + Repeat); on a correct
  match switch Pip to `cheer` and pop the correct choice image ("word comes
  alive"). All motion reduced-motion-guarded; the completion event is unchanged.
- `src/styles/child-ui.css` ‚Äî `.phonics-character` panel styles, the
  `is-speaking` / `is-cheering` / `is-alive` keyframes, and their entries in the
  existing `@media (prefers-reduced-motion: reduce)` `animation: none` list.
- `tests/modules/phonics-match.test.ts` ‚Äî behavioral coverage: Pip renders with
  the mouth for the activity's target sound; a correct match flips Pip to `cheer`
  and marks the correct choice `is-alive`.
- `package.json` / `package-lock.json` ‚Äî version bump to `0.3.18`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- The `ActivityAttemptEvent` shape and emission (correct/incorrect/hint/completed)
  ‚Äî Pip is presentation only; the same events fire at the same times.
- Grid lock, hint-after-N, correct-choice resolution, the chain "Next word"
  button ‚Äî all unchanged.
- `src/modules/tap-choice/TapChoiceActivity.ts` (Math/Shapes) ‚Äî untouched.
- The activity JSON schema and content ‚Äî no new fields required; Pip reads the
  existing `content.target_sound`. Activities without it fall back to `idle`.
- Curriculum graph, mastery, transfer, recommendation, parent lanes, storage ‚Äî
  untouched.
- **No reward loop**: the cheer + image pop is a single deterministic reaction to
  a correct answer (no points, streaks, randomness, or repeat-to-win incentive),
  consistent with the safety contract.
- Bear Cafe art (`bear-art.ts` etc.) ‚Äî untouched; Pip is a separate character in
  a separate module (distinct coral palette).
