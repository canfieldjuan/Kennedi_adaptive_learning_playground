# v0.3.4 Bear Cafe Delivery Handoff Beat

First slice of issue #3 (game richness backlog) applied to Kennedi's Orders.

## Before Code

### Root Cause

The delivery flow has no "in-between" beat: tapping `Deliver it` swaps the
delivery stage to the static complete screen instantly
(`KennedisOrdersActivity.ts` render loop). The plated food the child assembled
never visibly travels to the bear, and the bear never reacts to receiving it —
so the game renders end-states and skips the verb. Issue #3 names this the "No
delivery handoff beat" gap.

### Correct Fix Must Touch

- `src/modules/kennedis-orders/KennedisOrdersActivity.ts`
  - Add a transient `handoff` view stage between `delivery` and `complete`.
  - On `Deliver it`, go to `handoff` (not straight to `complete`).
  - The `handoff` stage renders a scene where the plated food travels to the
    bear and the bear does a receive/happy bounce, then after a fixed beat
    advances to `complete` (running the existing completion logic: chime,
    happy line, `emitCompletedEvent`).
  - Clear the timer on teardown (existing `cleanupHandlers`).
- `src/styles/child-ui.css` — scoped `.bear-cafe-handoff*` styles + two
  keyframes (`cafeHandoffTravel`, `cafeHandoffReceive`), added to the existing
  `prefers-reduced-motion` guard.
- `package.json` / `package-lock.json` — version bump to `0.3.4`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- No change to the evidence model: the same `order_delivered` completion event
  fires (just after the handoff beat instead of on the click). No new event
  outcomes or metadata.
- No change to activity JSON, activity chain, schema, mastery, transfer,
  recommendation, storage, export/reset, parent panel, home grid, or the
  child gate.
- No new games, no Nature Camera Safari (not present on main), no backend,
  rewards, streaks, or automatic routing.
- Motion stays preschool-safe: calm, short, reduced-motion-guarded.
