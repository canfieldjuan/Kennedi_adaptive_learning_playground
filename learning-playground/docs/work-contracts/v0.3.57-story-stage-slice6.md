# v0.3.57 Kennedi's Story Stage — slice 6: story history and parent review

A narrowly scoped, non-evaluative local record for story sessions (spec
§21), a Recent Stories parent section, export inclusion, clear-data
inclusion — and §20 held structurally: no correctness, no mastery, no
trait inference anywhere in the record or its presentation.

## Before Code

### Root Cause (slice)

Stories currently vanish when the route changes: a parent has no way to
see what was told, and export/clear don't know stories exist. §21 wants
a session record that is deliberately NOT an attempt event — the
existing event contract must not be distorted to pretend a creative
decision has a correct answer, so this is a separate record type, a
separate storage key, and a separate sink the runtime writes to.

### Required Change Surface

- `src/types/story-history.ts` (new) — `StoryHistoryRecord`: exactly
  §21's allowed fields — `story_session_id`, `mode`
  (`narrated`|`together`), `family_id`, `character_id`, `setting_id`,
  `problem_id`, `choice_path: string[]`, `ending_id?`, `started_at`,
  `completed_at?`, `status: 'completed' | 'left_early'`. Nothing else —
  no narration, no audio, no interpretations, no scores.
- `src/core/storage.ts` — `lp_story_history` key;
  `getStoryHistory()` (validated per record via a type guard, the
  parent-observation pattern; junk entries dropped),
  `appendStoryHistory(record)`, `clearStoryHistory()`;
  `exportProgressData` passes `storyHistory`.
- `src/core/export-data.ts` — payload/params gain
  `storyHistory?: StoryHistoryRecord[]` → `story_history` section +
  its entry in `data_sections_included`.
- `src/core/story-history.ts` (new, pure, DOM-free) —
  `buildRecentStories(records, pack, limit = 10)`: newest-first
  parent-readable items — title ("Princess Poppy at The Enchanted
  Forest"), problem label, mode label, status label ("Completed" /
  "Left early"), date — labels resolved from the pack passed as a
  PARAMETER (core never imports module content); unknown ids fall back
  to the raw id. No trait language anywhere.
- `src/modules/story-stage/StoryStageActivity.ts` — options gain
  `history?: { append(record: StoryHistoryRecord): void }` (a narrow
  sink — still no event log, no storage service):
  - Start → open a pending record (session id via
    `crypto.randomUUID()` with the established fallback, mode,
    family + selection ids, `started_at`, empty choice path).
  - Decision taps append choice ids to the path.
  - Reaching an ending → `status: 'completed'`, `ending_id`,
    `completed_at`, sink.append, pending cleared.
  - Tell It Again → a fresh pending record (same selection, new
    session id / started_at).
  - Leaving mid-story (route change / Home → `destroyStoryStage`, or
    New Story is unreachable mid-story) → the pending record flushes
    as `left_early` (no `completed_at`).
- `src/app/main.ts` — wires
  `history: { append: (record) => storage.appendStoryHistory(record) }`.
- `src/modules/parent-panel/ParentPanel.ts` — **Recent Stories**
  section (after Story Stage Mode): up to 10 items from
  `buildRecentStories(storage.getStoryHistory(), FIRST_STORY_PACK)`,
  each "title — problem" + "mode · status · date" meta; empty state
  line when no stories; the Clear Progress action additionally calls
  `clearStoryHistory()`.
- `src/styles/parent-ui.css` — recent-stories list styles.
- Tests —
  - `tests/contract/story-history.test.ts` (new): round-trip
    persistence + append order; the guard drops malformed entries;
    export payload contains `story_history` + section listing; clear
    removes records; `buildRecentStories` labels/order/cap/empty +
    left-early wording; the record type carries NO correctness or
    evaluation keys (structural sweep over stored JSON).
  - `tests/modules/story-stage.test.ts`: completed record shape (full
    choice path, ending id, mode, timestamps) for narrated AND
    together runs; left_early flush on destroy mid-story; Tell It
    Again produces a second completed record with a distinct session
    id; no record appended before an ending while playing; setup-only
    visits append nothing.
- `package.json` / `package-lock.json` — `0.3.57`; `MVP_BASELINE.md`.
- Captures — Recent Stories section with a few records (desktop +
  390×844).

### Explicit Non-Scope / Must Not Change (slice 6)

- NO ActivityAttemptEvents, no correct/incorrect/hint outcomes, no
  mastery/transfer/difficulty evidence, no imagination scores, no
  trait inference (§20/§21) — the record and its rendering carry
  facts only.
- No adult narration, audio, or child speech stored.
- Story child flow byte-identical (recording is invisible); setup
  produces no records.
- Pack/resolver/validator/selection modules at 0 lines; protected
  games at 0 lines; home untouched.

### Assumptions / Blockers

- "Interrupted" (§21) = leaving the STORY phase before an ending;
  abandoning setup records nothing (no story existed).
- Recent Stories caps at 10 in the panel; storage keeps all records
  (export completeness) — a retention cap is future hardening, noted
  not built.
- Blockers: none known.
