# v0.3.3 Bear Cafe Child-Started Play Review Contract

## Before Code

### Root Cause

Bear Cafe now starts from the child home grid, so parent review must explain the child-started order flow. The Bear Cafe runtime already emits `completed` delivery events with prompt text, selected tray, correct order, response time, hint state, and outcome. The parent Recent Attempts formatter excludes `completed` events, so the parent can see a completed activity summary but can miss the delivered-order evidence that explains what actually happened in the Cafe shift.

### Correct Fix Must Touch

- `src/core/parent-review-format.ts` to include completed activity events in Recent Attempts.
- `tests/contract/parent-review-format.test.ts` to prove completed events remain parent-readable and Bear Cafe delivered orders show friendly title, skills, prompt, selected answer, correct answer, hint usage, response time, and outcome.
- `docs/work-contracts/v0.3.3-bear-cafe-play-review.md` to record the before-code contract and final audit.
- `CURRENT_STATUS_AND_ROADMAP.md` and `MVP_BASELINE.md` to update the current product state.
- `package.json` and `package-lock.json` to bump the app version if verification passes.

### Must Not Change

- Do not change the Bear Cafe runtime, activity JSON, activity chain, child home grid, Video Vault direct route, event schema, activity schema, storage schema, export/reset behavior, mastery engine, transfer coverage engine, recommendation engine, parent approval semantics, or Vercel config.
- Do not add new game modules, new activities, `Nature Camera Safari`, backend services, auth, cloud sync, AI tutor chat, open web content, rewards, streaks, leaderboards, public sharing, or automatic adaptive routing.
- Do not weaken safety, parent approval, transfer truth, mastery, or local-first data rules.
