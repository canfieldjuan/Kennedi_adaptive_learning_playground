# v0.3.22 Voice tuning — warmer TTS + parent voice picker

Per the Voice Contract (local-first, no cloud/model voice, parent-can-disable),
tune the child-mode voice: warmer browser-TTS defaults and a parent-lane picker to
choose from the device's installed speech voices, with a "Test voice" button. Stays
browser speechSynthesis (on-device); no clip system this slice.

## Before Code

### Root Cause

The child voice is browser `speechSynthesis` with fixed defaults (rate 0.85 / pitch
1.1 / en-US) and **whatever voice the browser picks** — the parent cannot choose the
voice (`speech.ts`; `ParentSettings` has only `speech_enabled`). Different devices
default to very different voices, some not warm. There is no way to select or
preview a voice.

### Correct Fix Must Touch

- `src/core/speech.ts` — `SpeechService` accepts a preferred `voiceURI`
  (constructor + `setVoiceURI`); in `speak()`, resolve the matching
  `SpeechSynthesisVoice` from `getVoices()` and set `utterance.voice` (unchanged
  when unset/unavailable). Export pure helpers `pickVoice(voices, uri)` and
  `listSpeechVoices()` (English voices; `[]` when speechSynthesis is absent) for the
  picker and for testability. Keep warm, short-prompt behavior.
- `src/types/storage.ts` — add optional `speech_voice_uri?: string` to
  `ParentSettings` (undefined = device default; backward-compatible with stored
  settings).
- `src/modules/parent-panel/ParentPanel.ts` — a new "Voice" settings section
  (mirroring `createParentGateSettingsSection`): a `<select>` of installed voices
  ("Device default" + each voice), a "Test voice" button that previews the selected
  voice with a short sample, and Save → `storage.saveSettings({...settings,
  speech_voice_uri})` + re-render. **Guarded**: when `listSpeechVoices()` is empty
  (tests/SSR/no speechSynthesis), render "Device default" only + a note — never throw.
- `src/app/main.ts` — pass `settings.speech_voice_uri` into `new SpeechService(...)`.
- `tests/core/speech.test.ts` (new) — `pickVoice` selects by URI / returns undefined
  when absent; `listSpeechVoices()` is `[]` without speechSynthesis.
- `package.json` / `package-lock.json` — version bump to `0.3.22`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- **Voice source stays local browser TTS** — no cloud, no model voice, no clip
  system, no mic/recording (that is Option B, not this slice). Honors the Voice
  Contract's no-cloud/no-AI rules.
- `speech_enabled` disable path — unchanged; when speech is off, nothing speaks
  (the picker/test only affect which voice is used when enabled).
- The `AudioService` sound-cue layer — untouched (separate from voice).
- The Sound/Speech/Video **summary** rows are display-only today; this slice does not
  add toggle-editing for them (out of scope) — only the Voice picker.
- Every voice prompt keeps its on-screen visual equivalent (unchanged prompts).
- Existing `ParentSettings` consumers — the new field is optional, so stored/older
  settings remain valid.
