# v0.3.33 Visual enrichment arc — audit + art-direction contract (stage 1)

First deliverable of the visual enrichment arc (issue #55). Documentation
only: the shared art-direction law and the per-screen visual audit that every
game pass must trace to. No assets are drawn in this slice, per the arc rule
("do not start by drawing assets without this audit").

## Before Code

### Root Cause (arc)

The visual system illustrates individual objects but does not compose them
into complete game environments with foreground, middle ground, background,
state transitions, and a recognizable identity. Verified by a live capture
pass: the home screen has the only real scene (the cafe backdrop) while its
own four cards remain dark-navy-with-emoji; Words/Art/Number Train float
warm illustrated objects on the dark app background; Bear Cafe overrides to a
pale gradient and its delivery stage is a bell emoji, a basket-emoji-labeled
button, and the customer floating in a bordered square.

### Correct Fix Must Touch (this slice)

- `docs/contracts/game-environment.contract.md` (new) — the shared
  art-direction contract: style, scene layers, hierarchy, motion,
  implementation/mobile/performance constraints, and the visual + automated
  verification bar every game PR must meet.
- `docs/visual-environment-audit.md` (new) — the per-screen audit (home,
  three Words modes, six Bear Cafe stages, Art, legacy Math, Number Train,
  direct-route Shapes, Video Vault/Puzzle status) with, for each screen:
  current primary visual/background/middle/foreground, weak areas, missing
  context, mobile constraints, reusable assets, the proposed environmental
  concept (scene map), and protected interaction behavior — plus the
  per-game PR delivery order.

### Must Not Change

- Any application code, styles, assets, tests, or content — this slice is
  docs-only. Version stays 0.3.33 (doc-only slices follow the established
  no-bump convention).
- `scripts/check-work-contract.mjs` — the new contract file is deliberately
  NOT registered there (registering docs is that script's owner's call; out
  of scope here).
- The audit's protected-behavior lists bind the later game passes:
  curriculum, difficulty, evidence, mastery, routing, game rules, activity
  outcomes, parent behavior, event timing/commit points.
