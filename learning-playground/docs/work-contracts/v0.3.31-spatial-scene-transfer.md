# v0.3.31 Spatial Scene Transfer

## Before Code

### Code Read Cold

- `shapes-find-circle` and `shapes-find-circle-heart` both ask the child to find
  a named circle from isolated shape cards. They share `tap_to_match`, the
  `tap-shape-recognition` context, `prompt_mode: mixed`, and weak transfer
  context types; only the distractor set changes.
- The `shape_match` curriculum ladder describes finding target shapes and
  finding shapes across changes. It already plans `different_prompt_mode`, and
  its highest rung accepts difficulty levels 4-5.
- The content-gap engine already recommends the machine template
  `find_shape_in_scene` and parent pattern `Find Shape in Scene` for a spatial
  `different_prompt_mode` gap.
- The generic tap-choice runtime already renders local prompt images and emits
  selected/correct ids, response time, attempt number, hint state, incorrect,
  correct, completed, and hint events. Its prompt-image layout assumes small
  square icons, so a full scene would be too small to inspect safely.
- No approved spatial activity references
  `brief-shape_match-different_prompt_mode`, and the transfer recommendation
  engine therefore has no medium spatial activity to offer after weak evidence.

### Root Cause

Spatial transfer is blocked by missing approved content and an icon-sized visual
surface, not by missing mastery or event logic. The only approved evidence asks
for the same circle through the same isolated-card prompt. Adding another shape
card would remain weak; labeling a tiny scene image medium would make the
metadata look stronger without giving the child a legible contextual task.

### Correct Fix Must Touch

- Add one local illustrated scene asset with a clearly triangular roof, plus
  non-target circular and square objects, so the target is contextual rather
  than an isolated shape card.
- Add an optional `scene` prompt-visual layout to `TapChoiceActivity`. Only
  activities that explicitly request that layout and provide one prompt image
  may receive the scene classes; existing tap-choice rendering and event
  behavior must remain unchanged.
- Add scoped CSS that makes the scene large enough to inspect in portrait and
  arranges scene plus choices without unsafe target shrinkage in phone
  landscape. Existing icon prompt visuals and choices must keep their current
  dimensions.
- Add one approved `shape_match` activity that asks which shape the roof is
  without saying "triangle" in the prompt, uses the existing tap-choice
  runtime, declares medium `different_prompt_mode`, references
  `brief-shape_match-different_prompt_mode`, targets a new correct shape rather
  than another circle, and remains parent-launchable outside home routing.
- Register the activity and extend rich-transfer truth validation so the scene,
  named object, target shape, prompt image, correct choice, and prompt wording
  must agree.
- Add focused tests proving curriculum placement, weak-only spatial coverage,
  recommendation of the approved scene activity, scene-layout scoping, parent
  approval, no external content, and unchanged generic prompt-image layout.
- Advance package, baseline, and roadmap to v0.3.31 and leave
  video/vocabulary breadth as the remaining transfer-content risk.

### Must Not Change

- `src/core/mastery-engine.ts`, `transfer-coverage.ts`, `evidence.ts`,
  `progress.ts`, and `adaptive-engine.ts`: no mastery, strength, promotion,
  evidence, retention, or threshold changes.
- `src/core/content-gap-engine.ts` and
  `transfer-activity-recommendation.ts`: their current spatial recommendation

- Activity, event, storage, transfer, and curriculum schemas; scene metadata

- `curriculum.v1.json`; it already declares the skill, rung band, and planned
  transfer context this activity fulfills.
- Existing shape activity JSON, assets, prompts, versions, completion behavior,
  and direct routes.
- Home routing and the four child tiles; the new activity is reachable only
  through the existing direct/parent-approved transfer route.
- Coloring, Number Train, Word game, Bear Cafe, Video Vault, speech/voice,
  parent gate, parent approval, persistence, and export behavior.
- No new game, reward, streak, timer, backend, auth, cloud sync, external
  content, dependency, child-facing AI, reading requirement, or pressure.
- The unrelated untracked `.github/workflows/` directory.

### Acceptance Standard

- Existing prompt-image activities do not receive scene layout classes.
- The new activity displays one legible local scene and three large shape
  choices without requiring the child to read the answer.
- The spoken/text prompt identifies the roof but does not name the triangle.

  scrolling.
