# v0.3.51 Kennedi's Story Stage — slice 1: contract + vertical story

The first slice of the storytelling game: one complete, fixed, fully
narrated story proves the whole child flow (scenes → decisions →
consequences → ending) before any content system is built.

## Before Code

### Root Cause (slice)

The playground has no storytelling runtime: no way to hear a complete
authored story, make narrative decisions inside it, and reach a real
ending. One parent improvises stories happily; the other needs the game
to carry the entire narrative. Nothing in the current app serves either.

### Architecture decision (load-bearing, made before code)

Story Stage lives OUTSIDE the activity/evidence system: its own route
(`#story-stage`), no catalog entry, no `skill_ids`. This is not a
shortcut — catalog activities' `skill_ids` feed the difficulty-coverage
producers, the transfer approved-context counts, and the recommendation
engine. A story "activity" claiming vocabulary (or any skill) would
silently turn creative play into academic producers. Keeping the game off
the catalog makes the non-evaluation rule structural: the runtime takes
`speech` only — it has no `onEvent`, no child/session ids, and cannot
emit attempt events even by accident.

### Required Change Surface

- `src/app/router.ts` — `Route` union + parse + navigate gain
  `{ view: 'story-stage' }` (`#story-stage`). The four game routes and
  the home contract are untouched.
- `src/app/main.ts` — render/destroy wiring for the new view (speech
  service only).
- `src/modules/story-stage/` (new):
  - `story-stage.types.ts` — the smallest honest slice-1 types: a fixed
    story of typed scenes (`intro | problem | decision | consequence |
    ending`), two-choice decisions, per-choice `next` ids, ending flags.
    (Slice 2 replaces this with the full authored model + validator; the
    slice-1 story becomes migration input.)
  - `first-tale.ts` — the fixed Lost Friend tale: Princess Poppy
    (explorer) + the enchanted forest + Biscuit the missing-but-safe
    puppy. Eight authored scenes, six-scene play path, two real
    decisions: WHERE to look (sparkly path vs friendly owl → two
    distinct consequence scenes) and HOW to help Biscuit come out
    (gentle bubbles vs a soft song → two ending variants). Branches
    converge at the second decision only AFTER each first-choice branch
    has narrated its own visible consequence. Emotional arc: concern →
    curiosity → reassurance → cooperation → safe reunion; Biscuit is
    explicitly safe from the first mention.
  - `story-art.ts` — local inline SVG in the illustrated standard: the
    forest scene, Poppy (happy / worried-but-safe / celebrating faces),
    Biscuit, and four choice illustrations (sparkly path, owl, bubble
    wand, song notes). Render-checked. No text in art, no emoji.
  - `StoryStageActivity.ts` — the runtime shell: icon-only Home/Repeat
    (the studio's noise-reduction pattern; title sr-only), the
    illustrated scene, a one-line narration caption (by design readable
    text — the adult can read it when speech is off), a neutral
    six-dot story path (beginning→end, no outcomes, no pressure), two
    large illustrated choice cards at decisions, a large Continue
    control otherwise (no autoplay between scenes), and an ending state
    with "Tell it again" + Home. Narration: one short beat per scene via
    the existing speech service; scene changes `stop()` then `speak()`;
    Repeat replays the current scene; destroy stops speech.
- Parent launch: a third Parent-Started Games card ("Story Stage — Tell
  Me a Story") routing to `#story-stage`.
- CSS — story-stage block (warm stage surface, scene-dominant layout,
  caption strip, choice cards, path dots, phone rules ~390×844 with the
  scene and both choices reachable without ping-pong scrolling).
- Tests —
  - `tests/modules/story-stage.test.ts`: the full narrated flow on both
    first-choice branches (deterministic: same choices → same scenes);
    decisions have exactly two choices with distinct next scenes; every
    path reaches an ending (inline graph walk of the fixed tale);
    Repeat replays; scene changes stop speech; Home/destroy stops
    speech; ending reached exactly once per run; "Tell it again"
    restarts; the runtime emits no events (its options have no event
    sink); decorative layers aria-hidden; no `http` in story data.
  - `tests/contract/`: `#story-stage` parses + navigates; the home grid
    still has exactly four cards (existing pin re-run proves no home
    change); Words/Cafe/Math/Art routes byte-identical (existing pins).
- `docs/contracts/story-stage.contract.md` (new, unregistered) — the
  product contract: two modes (narrated default / Tell It Together),
  non-evaluation rules, safety tone rules, copyright boundary, the
  future seams (authored model, parent setting, story history,
  parent-recorded narration, private character packs, child-entry
  proposal).
- `package.json` / `package-lock.json` — `0.3.51`; `MVP_BASELINE.md`.

### Explicit Non-Scope / Must Not Change (slice 1)

- No home-screen change (no fifth tile; the child-entry proposal is
  slice 7's deliverable).
- No story model/validator (slice 2), no Pick Three setup (slice 3), no
  extra families/content pack (slice 4), no Together mode or parent
  setting (slice 5), no story history/export/clear (slice 6), no
  story-room enrichment (slice 7).
- No attempt events, no skill/mastery/transfer/difficulty evidence, no
  correct/incorrect/hint outcomes anywhere in the runtime.
- No generative AI, cloud, TTS beyond the existing local speech service,
  microphone, recording, remote assets, external links, accounts,
  scores, streaks, timers.
- No copyrighted characters — original archetypes only.
- Words / Bear Cafe / Art / Math / Number Train / parent gating /
  mastery / transfer / progress: 0 lines.

### Assumptions / Blockers

- A dedicated route view is the correct registration for a
  non-evaluative game (rationale above); the parent-launch card pattern
  accepts an arbitrary route.
- The fixed tale is a valid slice-2 migration input; its shape
  deliberately previews the authored model without claiming to be it.
- Blockers: none known.
