# v0.3.43 Difficulty Coverage Truth

Issue #35 follow-up: distinguish a missing approved difficulty activity from a
child performance signal.

## Before Code

### Code Read Cold

- Progress promotion filters attempts through the current curriculum rung's
  difficulty band at `src/core/progress.ts:287-304` and requires five eligible
  attempts before promotion. A rung with no approved activity therefore cannot
  collect qualifying evidence.
- `validateCurriculumActivityCoverage` already detects missing rung coverage at
  `src/core/curriculum-graph.ts:178-212`, but the repository contract invokes it
  only for phonics skills at `tests/contract/curriculum-graph.test.ts:17-19` and
  `tests/contract/curriculum-graph.test.ts:92-98`.
- Parent Guidance builds mastery and transfer evidence at
  `src/core/parent-interpretation.ts:135-187`, but it receives no progress level
  state and exposes no difficulty-rung coverage.
- The Parent Panel has the progress profile at
  `src/modules/parent-panel/ParentPanel.ts:133-160`, displays a curriculum rung
  label separately at `src/modules/parent-panel/ParentPanel.ts:2458-2467`, and
  renders mastery/transfer evidence at
  `src/modules/parent-panel/ParentPanel.ts:1189-1200`; it never explains whether
  an approved activity can exercise the displayed rung.
- Issue #35's phonics coverage claim is stale. Current phonics activities cover
  every declared phonics rung, enforced at
  `tests/contract/curriculum-graph.test.ts:92-136`.
- Issue #35's content-brief claim is also stale. Blending and word-building
  templates are skill-aware at `src/core/content-gap-engine.ts:224-235`, with
  focused coverage at `tests/core/transfer-coverage.test.ts:228-345`.

### Current Coverage Reconstruction

Comparing `APPROVED_ACTIVITIES` to every declared curriculum band produces nine
real gaps:

- vocabulary level 2, difficulty 4-5
- counting level 1, difficulty 2-3
- subitizing level 2, difficulty 4-5
- numeral_recognition levels 1 and 2, difficulty 2-3 and 4-5
- quantity_construction levels 1 and 2, difficulty 2-3 and 4-5
- number_sequence levels 1 and 2, difficulty 2-3 and 4-5

Phonics and shape_match have full declared-rung coverage. The catalog is ground
truth for approved playable content; planned curriculum rungs are not proof that
an activity exists.

### Finding Classification

- **Contradicted - phonics existence:** current phonics skills are no longer
  capped by the old all-difficulty-2 catalog. Regraded and transfer activities
  cover every phonics rung.
- **Contradicted - content brief cause:** current blending and word-building
  content briefs are already skill-aware.
- **Confirmed - broader existence:** nine non-phonics curriculum rungs have no
  approved producing activity.
- **Confirmed - cause:** progression correctly requires current-band evidence,
  but coverage validation and parent explanation are scoped too narrowly. The
  app cannot distinguish "more evidence needed" from "no approved activity can
  produce that evidence."
- **Confirmed - severity:** P2 for parent guidance accuracy. A child can arrive
  at an uncovered rung, then the panel continues reporting performance and
  mastery guidance without disclosing that qualifying progress evidence is
  structurally unavailable.
- **Confirmed - fix direction:** derive coverage from the curriculum graph and
  approved catalog, pass the current progress rung into Parent Guidance, and
  report the content limitation explicitly. Regrading activities, weakening
  promotion, or inventing content would not reach this slice's truthfulness
  problem.

### Root Cause

The product has curriculum difficulty bands and band-gated promotion, but no
runtime model joining a child's current rung to the approved activity catalog.
Coverage is checked only for phonics in tests and is absent from parent
interpretation. Consequently, a missing-content condition is silently presented
as ordinary child progress instead of an app/content limitation.

### Correct Fix Must Touch

- Add `docs/contracts/difficulty-coverage.contract.md` defining catalog-derived,
  current-rung coverage and parent-only behavior.
- Add `src/core/difficulty-coverage.ts` with a deterministic evaluator that:
  - resolves the exact current curriculum rung;
  - lists approved activity IDs inside that rung's inclusive difficulty band;
  - reports covered and missing rungs across the skill ladder;
  - returns `covered` or `blocked_by_content_gap` for the current rung;
  - explains that a gap is a content limitation, not a child judgment;
  - returns no evaluation for an unknown skill or undeclared level.
- Update `src/core/parent-interpretation.ts` to accept optional current skill
  states and expose difficulty coverage without changing mastery or transfer
  evaluation.
- Update `src/modules/parent-panel/ParentPanel.ts` to pass the stored skill
  states and render a parent-only Difficulty Coverage evidence group with the
  current rung, inclusive band, approved activity count, playable rung count,
  missing rung labels, and plain-language reason.
- Generalize curriculum coverage contract testing to all skills while pinning
  the exact current gap set so additions/removals cannot drift silently.
- Add focused core tests for inclusive band boundaries, covered, blocked,
  unknown-skill, and unknown-level behavior.
- Add parent interpretation/UI contract coverage proving the actual stored
  current level controls the displayed coverage state.
- Advance package, baseline, and roadmap to v0.3.43.
