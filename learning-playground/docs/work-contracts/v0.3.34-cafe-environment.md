# v0.3.34 Bear Cafe environmental pass (visual arc stage 2)

Second deliverable of the visual enrichment arc (issue #55): Bear Cafe gets
its environment. Presentation only — one continuous illustrated cafe across
all six stages, the pickup-window delivery scene, and the removal of the last
gameplay emoji (☎ / 🔔 / 🧺). Traces to `docs/visual-environment-audit.md`
(Bear Cafe section) under `docs/contracts/game-environment.contract.md`.

## Before Code

### Root Cause (slice)

The game with the richest object art has the emptiest stages: `.bear-cafe`
overrides the app background to a pale gray gradient and every stage floats
isolated cards on it. The delivery stage is a 🔔 emoji chip, a Deliver button
whose visible label is the 🧺 emoji, and the waiting customer floating in a
blue-bordered square — no place, no visible order context. The home screen
got the cafe backdrop; the cafe itself got nothing.

## Contract Amendments

1. **The service window is the styled DOM element, not an SVG scene group.**
   The contract planned an SVG service-window group with an opening shutter;
   implementing it would coordinate-couple the DOM customer bear to SVG
   viewBox positions across responsive sizes (fragile on phones). Hardened
   instead: the existing `bear-cafe-delivery__window` / `bear-cafe-complete__
   bear` hooks are styled as the illustrated service window (sky glow, ink
   frame, counter sill) — same visual story, no coupling, and the motion
   budget stays on the existing (already reduced-motion-guarded) tray-travel
   beat, so **no new animation was added at all**. The `is-open` hook was
   therefore not needed.
2. **Serving-tray art re-cut after live capture review.** The first tray
   (100×100 viewBox, tan fills) squashed in wide slots and vanished against
   the counter band behind it. Re-cut with a tight `0 26 100 48` viewBox and
   cream-on-ink contrast; verified in a second capture pass.
