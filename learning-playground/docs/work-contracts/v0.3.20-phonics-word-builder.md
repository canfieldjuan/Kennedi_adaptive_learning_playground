# v0.3.20 Word game — word builder (tap letter tiles to spell)

Fifth slice of the Word-game build arc (issue #27). The top rung of the
pre-reading ladder: the child builds a pictured word by tapping letter tiles into
slots in order. A new interaction (tap-then-place) added as a **mode of the
Word-game runtime** (phonics domain), not a new top-level game type.

## Before Code

### Root Cause

The Word game can decode (blend sounds → word) but never lets the child *encode*
— build a word themselves. There is no word-builder interaction, no
`word_building` skill, and the tap-a-picture matcher can only pick a whole
choice, not assemble one from parts. Word-building is the arc's declared top rung.

### Correct Fix Must Touch

- `src/modules/phonics-match/WordBuilderActivity.ts` (new) — a self-contained
  `renderWordBuilderActivity` / `destroyWordBuilderActivity` runtime: picture +
  empty slots + scrambled letter tiles; tapping the correct next letter snaps it
  into the next slot; a wrong tap gives gentle feedback and, after N, a hint that
  highlights the right tile; completing the word cheers Pip and pops the picture.
  Reuses `phonics-character-art` (Pip) and emits the same `ActivityAttemptEvent`
  shape (incorrect per wrong tap; correct+completed when built). The merged
  matcher (`PhonicsMatchActivity.ts`) is left untouched.
- `src/app/main.ts` — import + teardown `destroyWordBuilderActivity`; a dispatch
  branch `interaction_model === 'tap_then_place' && domain === 'phonics'` →
  `renderWordBuilderActivity`, placed after the Bear Cafe branch (build activities
  set no `content.game`, so they skip it) and before the tap-choice fallthrough.
- `src/content/curriculum/curriculum.v1.json` — new `word_building` skill (phonics,
  prerequisite `blending`, non-overlapping difficulty bands 0-2/3-4/5-5 so every
  level is reachable, transfer contexts covering the activities); add
  `word_building` to `blending.unlocks`.
- `src/content/activities/build-{cat,dog,sun}.json` (new) — tap_then_place phonics
  word-builders over existing picture cards (cat/dog/sun, all 3-letter), chained
  build-cat → build-dog → build-sun (build-sun terminal).
- `src/content/activities/blend-bat.json` — add `next_activity_id: build-cat` so
  the ladder climbs from blending into word-building.
- `src/content/activity-catalog.ts` — register the three build activities after
  the blend block.
- `src/styles/child-ui.css` — `.word-builder` slot/tile styles + a one-time
  tile-snap animation, added to the `prefers-reduced-motion` `animation: none` list.
- `tests/modules/word-builder.test.ts` (new) — build in order completes + cheers +
  picture alive; a wrong tile gives incorrect + (after N) a hint highlighting the
  correct tile; slots fill left-to-right.
- `tests/modules/phonics-match.test.ts` — extend the Word-game chain test to run
  find-b → … → blend-bat → build-cat → build-dog → build-sun and terminate.
- `package.json` / `package-lock.json` — version bump to `0.3.20`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- `src/modules/phonics-match/PhonicsMatchActivity.ts` (the matcher) and
  `phonics-character-art.ts` — untouched; the word-builder is a separate runtime
  that reuses Pip's art.
- The activity JSON **schema** — `tap_then_place` is already an allowed
  `interaction_model`; tiles/slots/target_word ride the opaque `content` object.
- The Bear Cafe dispatch (`tap_then_place && content.game === 'kennedis-orders'`)
  — unchanged and still matched first; phonics builds carry no `game`.
- The `ActivityAttemptEvent` shape, storage, mastery/transfer engines — the
  builder emits the same event shape; `blending` / `initial_sound` /
  `letter_sound_match` skills unchanged (only `blending.unlocks` grows).
- `TapChoiceActivity.ts` (Math/Shapes) — untouched.
- **No new top-level game type, no auto-difficulty routing, no reward loop** — the
  builder is a phonics-domain activity mode; the chain is child-tapped and
  parent-approved; the completion cheer is a single deterministic reaction; every
  build activity keeps `requires_parent_approval: true`,
  `external_links_allowed: false`, no URLs.
