# v0.3.6 Bear Cafe Cook/Plating Beat

Third slice of issue #3 (game richness backlog) applied to Kennedi's Orders.

## Before Code

### Root Cause

A correct check jumps straight to the "Order ready!" delivery stage with no
assemble/plating moment: `onCheck` emits `tray_checked` then immediately sets
`stage = 'delivery'` (`KennedisOrdersActivity.ts`). Issue #3 names this "No
cook/plating beat" — there's no sensory moment where the order is prepared.

### Correct Fix Must Touch

- `src/modules/kennedis-orders/KennedisOrdersActivity.ts`
  - Add a transient `plating` view stage between a correct `make`/`fix` check
    and `delivery`.
  - On a correct check, go to `plating` (not straight to `delivery`); the
    `tray_checked` event still emits synchronously on the check, before the beat.
  - The `plating` stage renders the assembled plate with a plate-up animation,
    then after a fixed beat plays the "Order ready" chime + speech and advances
    to `delivery`. Clear the timer on teardown.
- `src/styles/child-ui.css` — scoped `.bear-cafe-plating*` styles + a
  `cafePlatingUp` keyframe, added to the `prefers-reduced-motion` guard.
- `package.json` / `package-lock.json` — version bump to `0.3.6`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- No change to the evidence model: `tray_checked` (correct/incorrect) still
  emits synchronously on the check — moved nothing into the timed beat, so an
  interrupted plating beat drops no evidence (the P2 lesson from the v0.3.4
  handoff). Only the cosmetic "Order ready" chime + speech now fire at the end
  of the beat instead of on the click.
- The incorrect/fix path is unchanged (still goes to `fix`, emits `hint_used`).
- No change to activity JSON, schema, mastery, transfer, recommendation,
  storage, parent panel, home grid, or the child gate.
- No new games, no Nature Camera Safari (not present on main), no backend,
  rewards, or automatic routing.
- Motion stays preschool-safe: short, calm, reduced-motion-guarded.
