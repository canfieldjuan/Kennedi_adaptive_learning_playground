## Before Code

### Root Cause

Story Stage is visually illustrated, but its production artwork exists only as
template-literal SVG assembled inside `src/modules/story-stage/story-art.ts`.
The Lost Friend scene therefore has no separately editable production source,
optimized local export, provenance record, or existing-versus-proposed visual
proof. The project cannot independently revise, audit, reuse, or approve that
scene through the canonical Inkscape workflow.

The cause is not missing gameplay or a broken story path. Runtime composition
predates the production-art workflow and currently owns both scene selection and
all illustration geometry in TypeScript.

### Correct Fix Must Touch

- `design-source/story-stage/story-stage-lost-log-proof.svg` (new): editable
  Inkscape source for one representative Lost Friend scene, authored from
  original basic shapes and paths with named scene layers.
- `public/assets/images/story-stage-lost-log-proof.svg` (new): optimized local
  SVG export with a stable `400 250` viewBox, no fonts, remote references,
  stock/library content, embedded data, or child instructions.
- `src/modules/story-stage/story-art.ts`: add the minimum visual hook that uses
  the local production export only for the exact representative combination
  `lost-log` + `poppy` + `forest`; every other scene/character/setting continues
  through the existing dynamic builders.
- `tests/modules/story-stage.test.ts`: prove the representative combination
  references the local decorative export, prove alternate character and setting
  selections still render their actual dynamic choices, and retain the existing
  no-event and `aria-hidden` boundaries.
- `docs/art/asset-provenance.md`: add a complete Category A proof-family entry
  with owner look approval explicitly pending.
- `docs/captures/story-stage/`: add existing and proposed desktop/mobile
  captures plus a contact sheet comparing the representative scene.
- This work contract: record the implementation, verification evidence, and
  final line-cited cold diff audit.

No package version, dependency, route, schema, storage, activity, or CSS change

### Art Direction Standard

- Show a readable story event before narration: Poppy has reached a friendly owl
  beside a hollow log, and Biscuit is visibly peeking from the opening.
- Preserve the established purple ink and warm flat-fill family while improving
  depth, silhouette variety, environment richness, character scale, and focal
  hierarchy. The scene should feel illustrated, not like objects on a card.
- Keep Poppy on the left, the log/Biscuit clue in the right foreground, and the
  owl above the log. Use a curved path and layered forest forms to guide the eye
  without resembling an interactive route or button.
- Maintain a safe `400x250` composition for the existing responsive stage crop.
  Poppy, Biscuit, owl, and the log clue must remain legible on phone and desktop.
- The art is decorative and non-interactive. Semantic narration and controls
  remain the source of accessible instructions.

  no third-party or generated geometry is incorporated.

- The complete third-party resource list is `None`; no unlisted stock, library,
