# v0.3.25 Number Train — foundation (arc slice 1)

First slice of the Number Train build arc — the first real Math game. This doc
carries the arc-level contract plus the slice-1 scope; later slices amend it.

## Before Code

### Root Cause (arc)

The application lacks a Math-specific gameplay and evidence layer. The Math home
tile routes directly to one generic one-question activity
(`HomeScreen.ts` → `#activity/math-count-stars-three`), rendered on the shared
`TapChoiceActivity` grid. All three registered Math activities (stars, hearts,
dot card) represent the quantity three; each is a single question that returns
home. There is no Math session, no progression, no structured-quantity visual
model (the dot card's `prompt_card` content is not even rendered by the generic
runtime), and no dedicated Math runtime. The curriculum conflates counting and
subitizing on the same activities (`skill_ids: ["counting","subitizing"]`), and
counting rung labels are Bear Cafe-flavored ("Counts small pretend-play sets",
"Counts in combined orders") rather than general Math rungs. The generic
one-question tap-choice runtime cannot represent structured quantities,
multi-round progression, number construction, or distinct Math skills.

### Arc shape (later slices, for context — NOT in this slice)

2 — structured multi-round session (deterministic seeded plan, journey
indicator, completion + replay); 3 — Load the Train (quantity construction with
add/remove/check); 4 — Missing Station (number sequence); 5 — curriculum
correction (distinct skills: counting / numeral recognition / quantity
construction / number sequence; subitizing bounded to small sets; general rung
labels; per-skill outcomes via the existing `skill_outcomes` event extension);
6 — child UI polish (illustrated cohesion, mobile viewport, reduced motion,
accessibility). Extension seams documented, not built: Math world menu, Number
Trail, Balance Garden, 21–50 range, comparison rounds.

### Explicit Non-Scope / Must Not Change

- Words/phonics runtimes, Bear Cafe behavior, Art activities, video vault —
  untouched.
- `TapChoiceActivity.ts` — untouched (no giant conditional Math engine); Shapes
  and the legacy Math activities still render on it via direct routes.
- The three existing Math activities and their JSON — preserved and reachable
  (parent transfer lane recommends `math-dot-card-three`; direct
  `#activity/math-*` routes still work). Not deleted because the home tile
  changed.
- Event storage keys, `ActivityAttemptEvent` contract, legacy event
  compatibility — unchanged (metadata is the existing optional field;
  `skill_outcomes` exists from #39 and is not needed for this single-skill slice).
- Curriculum graph — **unchanged this slice.** Known limitation (documented, not
  hidden): the count round also exercises numeral recognition, but no
  `numeral_recognition` skill exists yet, so slice-1 evidence maps to `counting`
  only and no subitizing evidence is emitted at all. The skill split is slice 5.
- Parent gate, home four-item structure, adaptive routing, reward systems
  (no stars/coins/XP/streaks/timers/leaderboards), cloud/backend/analytics.
- No speculative Math World architecture: no hub screen, no generic game engine,
  no plugin system, no empty interfaces for Number Trail / Balance Garden.

## Contract Amendments

- **Transfer context changed from `different_interaction_model` to
  `same_format_new_examples`.** `npm test` surfaced the transfer-context-strength
  contract (`validate-activities.test.ts`): rich transfer contexts require an
  `originating_brief_id` from the parent-approved activity-brief design queue,
  and `different_interaction_model` has no detectable-content validator, so
  implemented activities may not claim it at all; `parent-interpretation` also
  expects `different_interaction_model` to remain a *missing* counting context.
  The honest classification for the slice-1 count round is weak anyway — count a
  visible set and tap the matching numeral is the same format as the legacy Math
  activities, with new examples (structured train seats). A rich-context claim
  for Number Train (e.g. Load-the-Train as a genuinely different interaction)
  belongs to a later slice through the brief machinery, not by fiat here.
