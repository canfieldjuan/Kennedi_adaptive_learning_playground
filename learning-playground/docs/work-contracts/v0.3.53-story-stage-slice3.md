# v0.3.53 Kennedi's Story Stage — slice 3: Pick Three setup

The story setup flow: three short visual selection steps (character →
place → problem) with spoken labels, a visual summary with one large
Start control, compatibility-filtered options, and a fully narrated
launch into the resolved story. Spec §7 + §25/slice-3.

## Before Code

### Root Cause (slice)

Slices 1–2 auto-start a fixed selection: the child never picks the
three story ingredients that §4/§7 define as the heart of the setup
experience. The authored model from slice 2 already carries everything
selection needs (entities with `spokenIntro` + art keys, family
compatibility lists, a resolver keyed by selection) — the missing piece
is the child-facing Pick Three flow and the pure selection helpers that
keep offered combinations valid.

### Required Change Surface

- `src/modules/story-stage/story-selection.ts` (new, pure) —
  `selectableEntities(pack)`: the characters/settings/problems offered
  to the child, filtered to entities referenced by at least one
  family's compatibility lists (an authored-but-unwired entity must
  never render as a dead card); `findFamilyFor(pack, selection)`: the
  first family whose three compatibility lists contain the full trio,
  or `undefined`. These two make "valid compatible combinations"
  structural.
- `src/modules/story-stage/story-art.ts` — `CARD_ART` +
  `storyCardSvg(artKey)` for the three entity art keys (`poppy`,
  `forest`, `paw-prints`), composed from the existing builders; the
  problem card must communicate "a friend is missing" visually (paw
  prints + peeking Biscuit), never by text.
- `src/modules/story-stage/StoryStageActivity.ts` — two phases inside
  the one route view:
  - **Setup phase** (new, default): step heading spoken on entry
    ("Who is the story about?" / "Where does it happen?" / "What
    happens?"), large illustrated cards (tap → visual selected state +
    `aria-pressed` + spoken `spokenIntro`; never auto-advances), a
    large Next control disabled until the step has a selection, then a
    visual summary (three mini cards) + one large Start.
  - **Story phase**: Start resolves the picked selection through
    `findFamilyFor` + `resolveStory` and plays exactly the slice-2
    story flow (opening + intro spoken = fully narrated launch).
    Ending controls gain **New story** (back to setup, per §18)
    alongside Tell it again + Home.
  - No timers, no autoplay, stop-before-speak everywhere, destroy
    stops speech from either phase.
- `src/styles/child-ui.css` — setup styles (`story-stage__setup`,
  `__step-title` (visible, short), `__cards`, `__card`,
  `__card--selected`, `__card-art`, `__card-label`, `__next`,
  `__summary`, `__summary-item`, `__start`), preschool-sized tap
  targets, 390×844 behavior per §23. Conditional rendering (innerHTML
  swap) — no `hidden` attributes, so the `[hidden]`-vs-display trap
  cannot fire.
- Tests — `tests/modules/story-stage.test.ts`: setup-flow proofs (card
  tap speaks and selects without advancing; Next gated on selection;
  summary shows the three picks + Start; Start narrates opening +
  intro; New story returns to setup; destroy from setup stops speech),
  existing story-flow tests routed through a `startFirstStory` helper;
  pure-helper proofs with a synthetic two-family pack
  (`selectableEntities` filters unwired entities, `findFamilyFor`
  picks the supporting family / rejects unsupported trios).
- `package.json` / `package-lock.json` — `0.3.53`; `MVP_BASELINE.md`.
- Captures — `docs/captures/story-stage/` gains setup step + summary
  shots, desktop 1366×900 and mobile 390×844, WebP q78 (visual change
  ⇒ live capture pass before PR).

### Explicit Non-Scope / Must Not Change (slice 3)

- Pack content: NO new characters/settings/problems/families — with
  the slice-2 pack each step offers its single authored card; real
  choice arrives with slice 4's content pack. The selection MECHANISM
  is this slice's deliverable and must not be padded with throwaway
  entities (§9: no shallow assets to inflate combination count).
- No mode setting / Tell It Together surfaces (slice 5), no story
  history (slice 6), no home-screen change (slice 7 proposal).
- Slice-2 modules unchanged except where named: resolver, validator,
  pack types, first-tale data stay byte-identical.
- Everything in the arc's standing non-scope (no AI/cloud/mic, no
  event sink, no scores/timers/streaks, protected games at 0 lines).

### Assumptions / Blockers

- Step prompts and the summary line are UI strings (like slice 1's
  "What happens next?"), not pack narration — authored inline.
- Card tap speaks the entity's `spokenIntro` (authored in slice 2 for
  exactly this); §7's "speaks the character label" is satisfied since
  each spokenIntro contains the label.
- Blockers: none known.

## Contract Amendments

- **Progressive filtering instead of end-check.** The contract named
  `selectableEntities(pack)` + `findFamilyFor`. Implemented as
  `familiesSupporting(pack, partial)` with `selectableEntities(pack,
  partial)` derived from it: each step's cards are filtered by the
  picks already made, so a trio no family supports is unreachable by
  construction (the contract's intent, made stronger). `findFamilyFor`
  is `familiesSupporting(...)[0]`.
