# v0.3.22 Skill-Aware Phonics Engines

## Before Code

### Ground-Truth Finding Audit

Confirmed:

- The content-gap engine chooses a suggested template from only the skill domain
  and missing context. Its generic phonics fallback is
  `new_initial_sound_examples`, so blending and word-building gaps can produce
  briefs for the wrong skill.
- The curriculum graph validates level shape but does not compare a level's
  difficulty band with approved activities for that skill. It therefore accepts
  declared rungs that no shipped activity can exercise.
- Every approved `blending` and `word_building` activity is difficulty 2 while
  both skills declare later bands at difficulty 3-5. The progress engine
  correctly filters promotion evidence to the current rung, so those later
  rungs are unreachable from current content.

Contradicted or overstated:

- The issue claim that every phonics skill has only difficulty-2 content is
  false. `initial_sound` also has the difficulty-5 Bear Cafe category-sort
  activity.
- The issue claim that the progress algorithm itself is generally
  difficulty-blind is false in current main. It already checks the current
  curriculum difficulty band. Weakening that check would hide the content and
  curriculum mismatch instead of fixing it.

### Root Cause

Two independent engines are missing skill-aware truth:

1. Curriculum validation knows the declared graph but not whether approved
   activities can actually supply evidence for each declared phonics rung.
   Difficulty metadata and rung labels can drift apart without failing a
   contract.
2. Content-gap templates and game-family suggestions branch primarily on
   `domain`, treating all phonics gaps as initial-sound Bear Cafe work. New
   `blending` and `word_building` skills therefore receive recommendations for
   the wrong learning task and game surface.

### Correct Fix Must Touch

- `src/core/curriculum-graph.ts` — expose approved-activity coverage validation
  that reports every phonics skill level whose difficulty band has no approved
  activity carrying that skill.
- `src/content/curriculum/curriculum.v1.json` — make phonics rung labels and
  bands describe the approved chain actually shipped.
- Existing phonics activity JSON only — grade the later approved chain steps
  into their matching curriculum bands and bump each changed activity version
  so old local events keep their original difficulty semantics:
  - `phonics-banana-starting-letter`
  - `blend-hat`
  - `blend-bat`
  - `build-dog`
  - `build-sun`
- `src/core/content-gap-engine.ts` — choose templates, activity patterns, and
  game families from the concrete skill plus context. Initial-sound work may
  remain Bear Cafe; blending and word-building work must point to the existing
  Word game and exercise the named skill.
- `docs/contracts/curriculum-graph.contract.md` and
  `docs/contracts/activity-variant-brief.contract.md` — record activity-band
  reachability and the existing Word game as an allowed brief family.
- Focused curriculum/content-gap/progress tests — prove every approved phonics
  rung is reachable, changed activities are versioned, blending/building briefs
  do not fall back to initial-sound work, and current-rung promotion remains
  difficulty-gated.
- Version and baseline/status documents — identify this as v0.3.22 and align
  the roadmap with current main.

### Must Not Change

- `src/core/progress.ts` and `src/core/adaptive-engine.ts` — current-rung
  difficulty filtering, promotion thresholds, support thresholds, parent
  overrides, and level clamping stay unchanged.
- `src/core/mastery-engine.ts`, `src/core/transfer-coverage.ts`, and evidence or
  review scheduling — mastery, transfer strength, retention, and regression
  rules stay unchanged.
- Activity schema and event/storage schemas — no schema migration and no rewrite
  of historical events. Activity version bumps preserve changed difficulty
  semantics prospectively.
- Phonics and Word-game runtimes — no interaction, prompt, chain, artwork,
  feedback, or child-facing flow change.
- Bear Cafe, mobile layout, parent approval, safety, rewards, routing, backend,
  auth, cloud sync, and external-content boundaries.
- No new game, game module, or activity. This slice makes existing curriculum,
  metadata, and recommendations truthful.

### Acceptance Standard

- Every declared phonics rung has at least one approved activity for the same
  skill whose difficulty lies inside that rung's band.
- The validator reports both sides of the boundary: an uncovered rung fails and
  the shipped catalog passes.
- Regraded activities use new versions; unchanged activities keep their current
  versions.
- A blending gap recommends blending work in the Word game.
- A letter-sound gap recommends letter-sound work in the Word game.
- A word-building gap recommends word-building work in the Word game.
- An initial-sound gap keeps its current Bear Cafe/category-sort behavior.
- Progress still rejects evidence outside the current rung band.
- Tests, typecheck, build, lint if present, and `git diff --check` pass.
