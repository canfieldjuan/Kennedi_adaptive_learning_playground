# v0.3.21 Word game — card art cohesion

Sixth slice of the Word-game build arc (issue #27, the deferred polish pass).
Re-skins the Word-game picture cards into one cohesive illustrated style matching
Pip and the Bear Cafe (ink outline `#3a2461`, warm flat fills, friendly rounded
forms). Presentation only — no code, no logic, no schema change.

## Before Code

### Root Cause

The ten Word-game picture cards are visually inconsistent: some are flat with no
outline (bear, cat, sun, moon, top, bat) and some carry a dark near-black outline
(dog, fish, ball, hat); none use Pip's purple ink. The cat's ears also float
detached from its head. Against Pip's ink-outline illustration the cards read as a
grab-bag, so the Word game does not feel of-a-piece.

### Correct Fix Must Touch

- `public/assets/images/{bear,cat,sun,moon,top,dog,fish,ball,bat,hat}.svg` — redraw
  each in one cohesive style: ink stroke `#3a2461` (~3px, round joins/caps), warm
  flat fills, recognizable friendly forms, `viewBox 0 0 100 100`. Keep each
  subject the same (a cat stays a cat) and ImageMagick-safe (no nested
  `<g transform>` / 3-arg rotate). Render-verify the full set.
- `package.json` / `package-lock.json` — version bump to `0.3.21`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- The card **file paths** and subjects — activities reference the same `/assets/
  images/<name>.svg`; only the artwork inside changes, so every activity keeps
  working with no content edit.
- All runtime code, activity JSON, curriculum, schema, tests — untouched (pure
  asset swap; the cards are `<img src>`-referenced).
- Pip (`phonics-character-art.ts`) and the Bear Cafe art — untouched; the cards
  move toward their established style, not the reverse.
- Cards are used only by phonics/blend/build activities (verified: no other
  consumer), so no other game's look changes.
