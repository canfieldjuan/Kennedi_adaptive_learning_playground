# v0.3.39 Number Train — the friendly station (visual arc stage 5)

Fifth deliverable of the visual enrichment arc (issue #55): the Number Train
gets the station world the Math arc spec reserved. Presentation only. Traces
to `docs/visual-environment-audit.md` (Number Train section) under
`docs/contracts/game-environment.contract.md`.

## Before Code

### Root Cause (slice)

The train (good bones: illustrated engine, cars, passengers) floats on the
dark navy app background. There is no station, platform, track, or sky — the
journey strip is six plain gray dots, and numeral choices are dark cards. The
world the arc spec reserved ("counting at the platform, loading beside the
station, missing numbers along the track") does not exist visually.

### Correct Fix Must Touch

- `src/modules/number-train/station-environment.ts` (new) — the game-owned
  station scene: soft sky, **continuous** rolling hills and a **solid** town
  silhouette (no window groups), a station house with canopy and an
  **icon-only** sign (no readable text), a platform band, and a rail band
  drawn as continuous rails (no discrete sleeper ties). **Hard guardrail: no
  loose countable objects anywhere** — nothing in the scenery may read as a
  small countable set (no birds, clouds-in-groups, window rows, tie rows),
  because every round is a counting task. `createStationEnvironment()`
  returns the aria-hidden, `pointer-events: none` layer. Render-verified.
- `src/modules/number-train/NumberTrainActivity.ts` — container gains the
  `train-station` class; the environment mounts as the first child **inside
  `startSession` (after `container.innerHTML = ''`)** so it survives Play
  Again replays. No plan/seat/hint/event changes.
- `src/styles/child-ui.css` —
  - `.train-station` sky-day surface (light override of the dark app
    background), inert layer + stacking, ink text overrides
    (title/prompt/feedback/utility buttons — the shared white text is for
    the dark background);
  - the journey strip becomes a **route line with station markers** (line
    drawn behind the existing dots via background gradient; dots restyled as
    small station signs) — same DOM, same is-current/is-done semantics;
  - `.number-train__stage` gets a **platform band** under the train/path
    (background gradient band, no new elements); the arrival state
    (`--arrived`, existing class) warms the platform — CSS-only, no new
    animation;
  - numeral choice cards switch to the warm cream/ink treatment (the
    workshop pattern) so they stay the strongest elements on the light
    scene; **cars and occupied seats keep stronger contrast than all
    scenery** (scene inks capped below the car/seat border strength);
  - mobile: minor scene props hidden, platform/route chrome shallow,
    tap-target sizes untouched; no new animation anywhere.
- Tests — `tests/modules/number-train.test.ts`: the station layer renders
  once per session, is aria-hidden, contains no `<text>`, **and is present
  again (still exactly once) after Play Again** (the remount seam);
  existing behavioral suites stay green. `mobile-child-ui.test.ts`: inert
  layer + minor-prop hiding.
- `docs/captures/train-station/` (new) — count round + load round + arrival
  desktop, count round mobile ~390×844, WebP, for owner look approval.
- `package.json` / `package-lock.json` — version bump to `0.3.39`.
- Docs note in `MVP_BASELINE.md`.

### Explicit Non-Scope / Must Not Change

- Plan generation, seed/replay semantics, seat semantics, load controls,
  hint behavior, journey/arrival/replay events, entry-band difficulty,
  per-skill evidence — untouched (zero runtime logic changes; the scene
  keys off existing classes only).
- No countable objects in scenery (hard guardrail above); no readable text
  in the scene; no new animation (the existing arrive/pulse beats are the
  motion budget).
- The Number Train phone compaction block's existing sizing rules — additions
  only, no changes to existing selectors' values.
- No changes to other games, the legacy Math tap-choice activities, or the
  shared dark background used elsewhere.
