# v0.3.16 Phonics-match module at parity

First slice of the Word-game build arc (issue #27). Foundation only: a real
`phonics-match` runtime the Word game grows on, reproducing today's matcher
behavior with **zero behavior change**.

## Before Code

### Root Cause

The "Words" game has no runtime of its own ‚Äî the three phonics activities
(`interaction_model: "tap_to_match"`, `domain: "phonics"`) fall through the
router's default to `renderTapChoiceActivity` (`app/main.ts:176`), the generic
grid shared with Math and Shapes. `modules/phonics-match/` is a dead type stub
(no importers). There is nothing to build the real Word game on without either
forking the shared runtime (which also serves Math/Shapes) or continuing to
overload it.

### Correct Fix Must Touch

- `src/modules/phonics-match/PhonicsMatchActivity.ts` (new) ‚Äî a real
  `renderPhonicsMatchActivity` / `destroyPhonicsMatchActivity` runtime that
  renders the phonics matcher, reproducing the current behavior and DOM (same
  `activity-*` classes ‚Üí identical CSS/visual). Owns the Word game from here.
- `src/modules/phonics-match/phonics-match.types.ts` ‚Äî replace the unused stub
  with the types the runtime actually uses.
- `src/app/main.ts` ‚Äî add a dispatch branch: `interaction_model === 'tap_to_match'`
  **and** `domain === 'phonics'` ‚Üí `renderPhonicsMatchActivity`; everything else
  (incl. Math/Shapes `tap_to_match`) still falls through to `renderTapChoiceActivity`.
- `tests/modules/phonics-match.test.ts` (new) ‚Äî behavioral coverage the matcher
  never had (correct‚Üícompleted event; N-wrong‚Üíhint), via the existing mock-DOM
  pattern.
- `package.json` / `package-lock.json` ‚Äî version bump to `0.3.16`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- `src/modules/tap-choice/TapChoiceActivity.ts` ‚Äî unchanged; still serves Math
  and Shapes `tap_to_match` activities (scoped out by `domain !== 'phonics'`).
- The phonics activity JSON content ‚Äî untouched (parity: no content change).
- The event/attempt model, mastery, transfer, recommendation, parent lanes,
  storage ‚Äî unchanged (the new runtime emits the identical event shape).
- **Known no-ops stay no-ops this slice** (parity): `required_correct_count` and
  `feedback_rules.*.animation` remain unread, exactly as today. They are targets
  for later slices, not this one.
- No curriculum/schema change (no new content shape yet).
