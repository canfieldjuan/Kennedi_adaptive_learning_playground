# v0.3.1 Parent Bear Cafe Launch Contract

## Before Code

### Root Cause

Kennedi's Orders / Bear Cafe is now integrated as a safe direct-route game, but it has no parent-controlled entry point. The child home grid intentionally stayed unchanged in v0.3.0, so the game is only reachable by knowing `#activity/kennedis-orders-banana-001`, which is too hidden for parent-led testing.

### Correct Fix Must Touch

- `docs/work-contracts/v0.3.1-parent-bear-cafe-launch.md` to record the before-code contract and final audit.
- `src/modules/parent-panel/ParentPanel.ts` to add a parent-only Bear Cafe launch section.
- `src/styles/parent-ui.css` only for compact parent-panel launch presentation if needed.
- `tests/contract/parent-game-launch.test.ts` to prove the launch stays parent-owned and the child home grid remains unchanged.
- Status/version docs and package metadata if verification passes.

### Must Not Change

- Do not add Bear Cafe to the child home grid.
- Do not replace Videos, Words, Math, or Art.
- Do not add or import Nature Camera Safari.
- Do not add a new game module or new activities.
- Do not change activity schema, event schema, mastery engine, transfer coverage engine, recommendation engine, storage, export/reset behavior, or parent approval semantics.
- Do not add backend, auth, cloud sync, AI tutor chat, open web content, rewards, streaks, leaderboards, public sharing, or automatic child routing.
- Do not weaken safety, parent approval, transfer truth, or mastery rules.
