# v0.3.10 Curriculum Ladder Alignment

## Before Code

### Root Cause

Parent-visible skill level is computed as a numeric progress counter, not as a
curriculum rung. `current_level` seeds from stored progress or activity
`difficulty_level`, promotes by a flat `+1`, and caps at `5`. The curriculum
declares skill `levels[]`, but those rungs have not carried difficulty bands,
have not been validated as ladders, and have not been used by progress or
Parent Panel display. That makes the number path-dependent and semantically
empty.

### Correct Fix Must Touch

- `src/content/curriculum/curriculum.v1.json`
  - Add difficulty bands to each skill level so activity difficulty can map to
    a declared rung.
  - Keep promoted rungs reachable by the approved activities that currently
    exist, even when adjacent difficulty bands need to overlap.
- `src/core/curriculum-graph.ts`
  - Validate non-empty, contiguous skill levels starting at `0`.
  - Validate difficulty bands are inside `0..5` and ordered.
  - Expose helpers for level lookup, lowest level, max level, and difficulty to
    level mapping.
- `src/core/progress.ts`
  - Seed new skills at the lowest curriculum rung.
  - Clamp stored levels to the declared curriculum max.
  - Promote only from attempts inside the current rung's difficulty band.
  - Count parent-approved one-step `promote_gently` attempts as current-rung
    promotion evidence instead of filtering them out solely because the runtime
    raised the effective difficulty by one.
  - Cap promotion at the declared curriculum max, not hard-coded `5`.
- `src/modules/parent-panel/ParentPanel.ts`
  - Show the curriculum rung label instead of a bare level integer.
- Focused tests for graph validation, progress seeding/promotion/clamping, and
  parent display.
- Version/baseline/current-status docs for v0.3.10.

### Must Not Change

- Do not merge numeric progress level with `MasteryStatus` in this slice.
- Do not change event schema, activity schema, storage keys, export/reset,
  mastery rules, transfer coverage, recommendations, parent approval, Bear Cafe
  runtime, activity chain, home grid, backend/auth/sync posture, rewards,
  streaks, or automatic routing.

## Cold Diff Audit

### Gaps

- change without contract trace: none.

### Change By Change Reconstruction

- `curriculum.v1.json` adds `min_difficulty_level` and
  `max_difficulty_level` to each skill rung, and expands ladders where existing
  activities already use richer difficulty levels. Review follow-up keeps
  `letter_sound_match`, `subitizing`, and `shape_match` promoted rungs reachable
  from the approved activity difficulties that exist today.
- `curriculum-graph.ts` types level difficulty bands, validates ladder shape and
  bands, and exposes level/difficulty lookup helpers.
- `progress.ts` loads the curriculum graph while rebuilding local progress,
  seeds new skill levels from the lowest curriculum rung, clamps stale stored
  levels to the declared max, filters promotion evidence to the current rung's
  difficulty band, counts parent-approved one-step `promote_gently` attempts
  toward the current rung, and caps promotion at the declared max.
- `ParentPanel.ts` formats the progress level as a curriculum label.
- Contract tests cover activity difficulty to rung mapping, ladder validation,
  progress seeding, band-limited promotion, parent-approved one-step challenge
  evidence, max-level clamping, completion-only behavior, and parent-visible
  level labels.
- Version history records this as the v0.3.10 ladder slice; the package version
  is superseded to `0.3.11` by the bundled storage-migration slice.
- `MVP_BASELINE.md` and `CURRENT_STATUS_AND_ROADMAP.md` record the aligned
  curriculum ladder behavior.

### Contract Traceability

- Curriculum ladders become load-bearing through graph helpers and progress
  usage.
- Path-dependent seeding is removed because new progress starts from the
  lowest curriculum rung.
- Difficulty-blind promotion is removed because only attempts in the current
  rung's difficulty band, or parent-approved one-step challenge attempts,
  qualify.
- Parent display no longer exposes an unexplained raw number as the primary
  level.
- Mastery, transfer, schemas, parent approval, storage keys, and Bear Cafe
  runtime remain outside this change.
