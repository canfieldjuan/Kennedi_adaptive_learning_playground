# v0.3.38 Art — the little studio (visual arc stage 4)

Fourth deliverable of the visual enrichment arc (issue #55): Art becomes a
small creative studio. Presentation only. Traces to
`docs/visual-environment-audit.md` (Art section) under
`docs/contracts/game-environment.contract.md`.

## Before Code

### Root Cause (slice)

Art is a white circle and four swatch cards floating on the dark app
background — a lab, not a studio. There is no sense of making: no canvas, no
easel, no supplies, no place, and no state response when the child fills the
shape beyond the fill color itself.

### Correct Fix Must Touch

- `src/modules/coloring-book/studio-environment.ts` (new) — the game-owned
  studio scene, deliberately **all-neutral** (creams, wood tones, gray-beige):
  soft studio wall, pinned child-artwork shapes (abstract, neutral — never
  colored), a shelf with jars/brushes (neutral bristles), window, floor mat.
  **Hard guardrail: the scene contains no saturated color anywhere — color
  belongs exclusively to the palette swatches and the filled shape**, so no
  decoration can be confused with a color choice. `createStudioEnvironment()`
  returns the aria-hidden, `pointer-events: none` layer. Render-verified.
- `src/modules/coloring-book/ColoringActivity.ts` — container gains the
  `coloring-studio` class and mounts the environment first-child. No
  interaction, completion, or request-mode changes (the existing
  `is-complete` / `is-wrong` / `is-hinted` state classes already provide
  every hook the visuals need).
- `src/styles/child-ui.css` —
  - `.coloring-studio` warm-neutral surface + inert layer + stacking + ink
    text overrides (the shared white text is for the dark background);
  - the active shape framed as a **canvas on an easel** (white canvas panel
    with ink frame; easel legs below via scoped pseudo-elements);
  - the swatch row framed as a **supply tray** (wood-tone band); swatch
    chips keep their exact colors and sizes — contrast against the neutral
    scene is higher than before;
  - **state-responsive richness on existing classes**: `is-complete` adds a
    one-time paint-shine sweep on the filled shape (single keyframe,
    `prefers-reduced-motion` disabled) — the completed artwork stays
    displayed on the easel with the existing completion actions;
  - mobile: minor props hidden, tray/easel chrome shallow, the
    contract-tested coloring landscape grid untouched.
- Tests — coloring behavior suite: the studio layer renders once,
  aria-hidden, **no `<text>` and no saturated fill in the scene** (neutral
  guardrail asserted against a small named palette); existing coloring
  behavior tests stay green. `mobile-child-ui.test.ts`: inert layer +
  phone prop hiding.
- `docs/captures/art-studio/` (new) — start + filled desktop, start mobile
  ~390×844, WebP, for owner look approval.
- `package.json` / `package-lock.json` — version bump to `0.3.38`.
- Docs note in `MVP_BASELINE.md`.

### Explicit Non-Scope / Must Not Change

- Coloring interaction, completion logic, event emission, request-mode
  behavior, swatch selection semantics — untouched (zero runtime state
  changes; the scene keys off existing classes).
- No colored decorations anywhere near (or far from) the palette — the
  whole scene is neutral by design.
- The contract-tested coloring landscape grid CSS strings — byte-identical.
- No new dependencies, no readable text in the scene, no motion beyond the
  single one-shot shine (guarded).

## Contract Amendments

- **Version renumbered 0.3.37 → 0.3.38.** Codex's parent-observation-signals
  PR (#59) merged as 0.3.37 while this slice was in flight; origin/main was
  merged in and the slice renumbered (routine collision procedure, no code

  check cannot parse (known trap from the cafe pass); the frame is now
  straight. Caught by rendering, not by tests.
