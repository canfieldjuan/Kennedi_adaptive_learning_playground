# v0.3.17 Word game — more initial sounds

Second slice of the Word-game build arc (issue #27). Content + a small runtime
extension: turn the single-word matcher into a **multi-word session** across more
initial sounds, reusing the existing `phonics-match` runtime and tap-a-picture
mechanic. No new game type, no auto-difficulty routing, no reward loop.

## Before Code

### Root Cause

After slice 1 the Word game is one reachable activity from the child home
(`phonics-find-b`, all /b/). The other phonics activities (`phonics-banana-...`,
`phonics-find-b-ball`) are not on the home grid and there is no way to advance
from one word to the next inside a session, so the child burns through the single
matcher and stops. The Word game needs more initial sounds *and* a way to string
them into one session — without adding a new game type, changing difficulty
routing, or introducing a reward loop.

### Correct Fix Must Touch

- `src/content/activities/phonics-find-{m,s,c,t}.json` (new) — four initial-sound
  matchers reusing the existing envelope: `domain: "phonics"`,
  `interaction_model: "tap_to_match"`, `skill_ids: [initial_sound,
  letter_sound_match]`, `transfer.context_type: "same_format_new_examples"`
  (same game, new words). Each carries `content.next_activity_id` to chain to the
  next word; the last (`find-t`) has none.
- `src/content/activities/phonics-find-b.json` — add `content.next_activity_id`
  (`phonics-find-m`) + `next_label` so the existing first word starts the chain.
- `public/assets/images/{moon,top}.svg` (new) — two new flat-style cards matching
  the existing clip-art phonics cards (`moon` for /m/, `top` for /t/); the other
  new targets/distractors reuse existing cards (bear, cat, sun, dog, fish, ball).
- `src/content/activity-catalog.ts` — register the four new activities in
  `APPROVED_ACTIVITIES`, ordered **after** `phonics-banana-starting-letter` (see
  Must Not Change: preserves the parent transfer recommendation).
- `src/modules/phonics-match/PhonicsMatchActivity.ts` — on completion, if the
  activity declares `content.next_activity_id`, render a "Next word" button that
  navigates to `#activity/<next>`; otherwise only Home (unchanged). Read via two
  small opaque-content readers (`getNextActivityId`/`getNextLabel`), same pattern
  as the existing `content.*` readers.
- `tests/modules/phonics-match.test.ts` — a behavioral test that a chained
  activity shows a Next button that advances the hash, and a test that the chain
  is complete, ordered, and terminates.
- `package.json` / `package-lock.json` — version bump to `0.3.17`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- `src/modules/tap-choice/TapChoiceActivity.ts` — untouched (Math/Shapes runtime).
- **Parent transfer recommendation output** — the transfer recommendation engine
  (`core/transfer-activity-recommendation.ts`) picks the first uncovered-context
  activity by `APPROVED_ACTIVITIES` order. The new activities are
  `same_format_new_examples`; ordering them **after** the reverse_mapping
  activity keeps `phonics-banana-starting-letter` as the recommended "rich
  transfer" activity, so the parent lane behaves exactly as before. (Whether
  new-examples *should* rank before reverse-mapping is a pedagogy decision
  deferred to a follow-up, noted in the PR — not decided by array position here.)
- No curriculum/schema change — `initial_sound`/`letter_sound_match` already
  exist and `activity_contexts` is representative (non-exhaustive), so new
  same-skill activities need no graph edit. `same_format_new_examples` is already
  a `planned_transfer_context` for both skills.
- No new game type, no auto-difficulty routing, no reward loop — the chain is a
  fixed, hand-authored, parent-approved sequence; every new activity keeps
  `requires_parent_approval: true`, `external_links_allowed: false`, no URLs.
- The event/attempt model, mastery, storage — unchanged (the runtime emits the
  identical event shape; the Next button only navigates).
- Known no-ops stay no-ops (`required_correct_count`, `feedback_rules.*.animation`).

## Contract Amendments

- Added `src/content/activity-catalog.ts` reordering to **Must Not Change /
  Correct Fix Must Touch**: `npm test` proved that inserting the new activities
  *before* `phonics-banana-starting-letter` flipped the parent transfer
  recommendation from `phonics-banana-starting-letter` (reverse_mapping) to
  `phonics-find-m` (same_format_new_examples). Root cause: the recommendation
  engine selects by array order. Resolution: order the new activities after
  banana to preserve the existing recommendation (behavior parity), rather than
  re-rank the engine (out of scope; deferred).
