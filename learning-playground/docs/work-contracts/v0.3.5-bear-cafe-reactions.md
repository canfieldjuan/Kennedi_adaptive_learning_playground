# v0.3.5 Bear Cafe Reaction States

Second slice of issue #3 (game richness backlog) applied to Kennedi's Orders.

## Before Code

### Root Cause

The bear renders the same static `content.character.icon` in every stage —
order card (`KennedisOrdersActivity.ts:370`), delivery window (`:614`), handoff
(`:566`), and complete (`:625`). It never reacts to the arc of the order, so the
design doc §16's "happy / neutral / waiting reactions" are unimplemented (issue
#3, "Bears never react").

### Correct Fix Must Touch

- `src/modules/kennedis-orders/KennedisOrdersActivity.ts`
  - Add a small `bearReactionAccent(reaction)` helper mapping a reaction
    (`waiting` / `receiving` / `happy`) to an accent glyph span.
  - Render a stage-appropriate accent alongside the existing bear icon:
    waiting while the order is prepared (order card + delivery window),
    receiving at the handoff, happy on completion.
- `src/styles/child-ui.css` — scoped `.bear-cafe-reaction*` styles + three
  keyframes (`cafeReactionWait`, `cafeReactionReceive`, `cafeReactionHappy`),
  added to the existing `prefers-reduced-motion` guard.
- `package.json` / `package-lock.json` — version bump to `0.3.5`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- No change to `content.character` or the activity JSON schema: reactions are
  derived from a fixed stage→glyph map in the runtime, not new content fields.
  The base `content.character.icon` still renders unchanged in every stage; the
  accent is additive.
- No change to the evidence model, event outcomes, or metadata; no change to
  activity chain, mastery, transfer, recommendation, storage, export/reset,
  parent panel, home grid, or the child gate.
- No new games, no Nature Camera Safari (not present on main), no backend,
  rewards, or automatic routing.
- Motion stays preschool-safe: small, calm, reduced-motion-guarded accents.
