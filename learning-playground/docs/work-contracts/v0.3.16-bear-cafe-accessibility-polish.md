# v0.3.16 Bear Cafe Accessibility Polish

## Before Code

### Root Cause

Bear Cafe choice controls already track selected state visually with
`data-selected`, but the control state is not exposed to assistive technology.
Food choices also show counts visually after selection, while the accessible
button label remains a generic "Choose X".

### Correct Fix Must Touch

- `src/modules/kennedis-orders/KennedisOrdersActivity.ts` — expose the same
  selected/count state used by the visual Bear Cafe choices through accessible
  button labels and pressed state.
- `tests/modules/kennedis-orders.test.ts` — cover the food count and
  selected/not-selected accessibility label helpers.
- `docs/work-contracts/v0.3.16-bear-cafe-accessibility-polish.md` — record the
  contract, audit, and verification.

### Must Not Change

- No Words, phonics, `tap_to_match`, shared activity dispatch, or phonics content
  changes; issue #27 is happening in a separate lane.
- No activity schema, approved activity content, mastery, transfer, recommendation,
  progress, storage, parent panel, safety, backend, deployment, dependency, or
  workflow changes.
- No gameplay behavior changes: food taps still add food, tray remove buttons
  still remove food, colors/decorations still select, and order checks stay the
  same.
