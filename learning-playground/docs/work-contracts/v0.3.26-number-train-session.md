# v0.3.26 Number Train — deterministic multi-round session (arc slice 2)

Second slice of the Number Train build arc (issue #43). Turns the single fixed
count round into a short, replayable, deterministic **train trip**: a seeded
six-round session of graduated Count-the-Train rounds, a journey indicator
(train stations, never points), a one-time arrival celebration, and Play Again.

## Before Code

### Root Cause (slice)

The slice-1 foundation plays exactly one fixed round (quantity 7) and returns
home. There is no session, no progression within a sitting, no journey
representation, no completion arc, and no replay — so the Math tile is still a
single worksheet, not a trip. The round generator contract exists
(`validatePlan`) but nothing generates varied rounds, and the runtime's
feedback/hint speech is hard-authored for quantity 7 so it cannot serve varied
quantities.

### Correct Fix Must Touch

- `src/modules/number-train/number-train.types.ts` — add
  `NumberTrainSessionConfig` (`seed`, `round_count`, `max_quantity`).
- `src/modules/number-train/round-plan.ts` — replace the slice-1 scaffolding
  `buildFoundationPlan()` with `buildSessionPlan(config)`: a **seeded, pure,
  deterministic** generator (mulberry32 PRNG — no `Math.random`, no clock).
  Six graduated count rounds: quantities ramp from an easy confidence start
  (~3) to a slight stretch near `max_quantity`, non-decreasing, jittered
  within the ramp by the seed; distractors are plausible neighbors (±1/±2)
  clamped to `0..max`, never duplicated, with the correct answer positioned by
  seed. The generated plan is passed through `validatePlan` and **throws on
  violation** (fail closed). Same seed + config ⇒ identical plan, reproducible
  in tests.
- `src/modules/number-train/NumberTrainActivity.ts` — session state machine:
  render round *i* (train rebuilt for the round quantity, choices rebuilt,
  per-round attempt/hint state); correct → the journey station fills and a
  large child-tapped **Next station** control advances (no auto-advance, no
  timers, no pressure); final round → one-time deterministic **arrival**
  (train arrives at the station, spoken arrival line, `completed` event) with
  large **Play Again** and **Home** controls. Play Again rebuilds a new valid
  plan from the next stable seed (`seed + replay_index` — still deterministic).
  A **journey indicator** (one station marker per round: done / current /
  ahead) sits above the train; no score, no text beyond the round prompt.
  Correct/hint speech is composed per round quantity (hints: count-along
  enumeration up to ten; above ten, "one full car is ten" then count on —
  teaching the tens structure). `incorrect`/sounds stay envelope-authored.
  Timeout handles are tracked and cleared on destroy.
- `src/content/activities/number-train.json` — content gains
  `round_count: 6`, `seed: 20260710`, `arrival_audio`; the static correct/hint
  speech becomes the generic fallback (runtime composes quantity-specific
  lines). No schema change (opaque content).
- `src/styles/child-ui.css` — `.number-train__journey` / `__station` states
  (done/current), arrival celebration state, all animations added to the
  `prefers-reduced-motion` `animation: none` list.
- `tests/modules/number-train.test.ts` — updated + extended: seeded
  determinism (same seed ⇒ deep-equal plans; different seeds differ); every
  generated plan across many seeds passes `validatePlan`, stays within
  bounds, and ramps non-decreasing; full-session flow (six correct rounds →
  journey fills → **`completed` emitted exactly once**); wrong-then-hint mid
  session; Play Again produces a new valid, different, playable session;
  destroy mid-session cleans up (re-render clean); structured-car assertions
  recomputed from the deterministic plan instead of the old fixed 7.
- `package.json` / `package-lock.json` — version bump to `0.3.26`.
- Docs note in `MVP_BASELINE.md`.

### Explicit Non-Scope / Must Not Change

- Round types: **Count the Train only** — Load-the-Train (slice 3) and
  Missing Station (slice 4) are not in this slice; the round union stays
  as-is until they land.
- Curriculum graph, skills, evidence semantics — unchanged (`counting` only,
  no subitizing; the skill split is slice 5). Difficulty stays 1 (entry band —
  the PR #44 review invariant and its test hold).
- No adaptive routing, no reward systems (journey = train position, not
  points/streaks), no timers pressuring the child, no unbounded randomness
  (seeded PRNG only), no new top-level activities per round.
- `TapChoiceActivity`, phonics runtimes, Bear Cafe, coloring, legacy Math
  activities, event contract/storage keys, home tile structure — untouched.
- No speculative Math-world architecture.
