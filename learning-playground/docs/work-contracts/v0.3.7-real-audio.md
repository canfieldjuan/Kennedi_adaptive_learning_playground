# v0.3.7 Real Audio (shared runtime)

Fourth slice of issue #3 (game richness backlog). Unlike the prior three, this
is a shared-runtime win, not a Bear-Cafe-only change: every game runtime calls
`audio.play(...)`, so wiring real audio once benefits all of them.

## Before Code

### Root Cause

`AudioService.play()` (`src/core/audio.ts`) only `console.log`s — the ring /
chime / boing cues the runtimes request (`audio.play('soft_chime' |
'soft_boing')`) are silent; only TTS speech is real. Issue #3 names this "Sounds
are stubbed."

### Correct Fix Must Touch

- `src/core/audio.ts` — synthesize the cues with the Web Audio API so
  `soft_chime` (success / order-ready / delivery) and `soft_boing` (try-again)
  actually sound, while keeping `stop()` and the `AudioServiceInterface`
  contract intact and honoring the `enabled` (`sound_enabled`) flag.
- `package.json` / `package-lock.json` — version bump to `0.3.7`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- **Safety boundary:** no external audio assets, no network, no dependencies —
  sounds are synthesized in-process, so "local-only / no open web" still holds
  and no `http(s)` is introduced.
- `AudioServiceInterface` (`play(soundId)` / `stop()`) is unchanged; construction
  (`new AudioService(settings.sound_enabled)`) and the parent Sound Effects
  toggle keep working. Sound stays off when `sound_enabled` is false.
- No change to any activity, schema, event/evidence model, or other runtime.
- Cues stay preschool-safe: soft, low-volume, short, non-startling.

### Assumptions

- The two cue ids actually played anywhere in the app today are `soft_chime`
  and `soft_boing` (verified by grep). Unknown ids stay silent rather than guess
  a sound.

## Cold Diff Audit

### Gaps

- change without contract trace: none.
- contract requirement not delivered: none.
- protected surface touched: none.

### Change By Change Reconstruction

- `src/core/audio.ts` — replaces the `console.log` placeholder with a Web Audio
  synth: a lazily-created, feature-detected `AudioContext` (no-op if the API is
  absent, e.g. tests/SSR), resumed on play to satisfy the autoplay policy;
  `soft_chime` = a soft two-note sine bell (A5 + E6), `soft_boing` = a gentle
  downward sine wobble; a shared enveloped `tone()` helper avoids clicks;
  `stop()` suspends the idle context. Still gated by `enabled`.
- `package.json` / `package-lock.json` — `0.3.7`.
- `MVP_BASELINE.md` — records real audio.

### Contract Traceability

- Audible cues → issue #3 "Sounds are stubbed"; delivered by `src/core/audio.ts`.
- Safety preserved → synthesis only; no assets/network/deps; no `http(s)`.

### Verification

- `npm run typecheck` — passed.
- `npm test` — 31 files / 150 tests passed (no audio test existed or broke; the

- `npm run build` — passed.
