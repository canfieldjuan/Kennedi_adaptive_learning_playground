# v0.3.50 Number Train upper bands — six content gaps filled

The "upper-band producers" seam from the Number Train arc (documented in
the v0.3.32 contract as described-not-built): two higher trips make every
declared rung of the three train skills playable. Content + data only —
the generator already validates plans to 50.

## Before Code

### Root Cause (slice)

Seven curriculum rungs have no approved activity, and the parent coverage
UI truthfully reports them as "Blocked by content gap". Six belong to the
train skills — `numeral_recognition`, `quantity_construction`, and
`number_sequence` at levels 1 (band 2-3) and 2 (band 4-5) — because the
only Number Train activity sits at difficulty 1. The 21–50 range was
designed for (NUMBER_TRAIN_ABSOLUTE_MAX = 50; cars of ten; validated
plans) but never given an activity. The seventh gap (`subitizing:2:4-5`)
stays: subitizing is deliberately fenced off from train evidence, and it
is the coverage UI's live demo gap (the CI browser spec pins it).

  the three level-1 (2-3) bands. Weak transfer

  difficulty-coverage tests and one parent-interpretation test pin the
  band's producer list as exactly `[art-studio-five-flowers]`; the
  Express legitimately joins it. The synthetic-gap test now removes BOTH
