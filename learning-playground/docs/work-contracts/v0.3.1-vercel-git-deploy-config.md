# v0.3.1 Vercel Git Deploy Config Contract

## Before Code

### Root Cause

The Vercel project config explicitly disabled Git-triggered deployments with `git.deploymentEnabled: false`. That makes the live Vercel site depend on manual deployments, so production can remain stale even when GitHub has newer reviewed work.

### Correct Fix Must Touch

- `vercel.json` to remove the explicit Git deployment disable flag while preserving the existing Vite build settings.
- `.gitignore` to keep local `.vercel` project metadata and pulled environment files out of source control.
- `docs/work-contracts/v0.3.1-vercel-git-deploy-config.md` to record the scope and final audit.

### Must Not Change

- Do not change child-facing app behavior, parent panel behavior, activities, content packs, mastery logic, transfer coverage, recommendation logic, event logging, or storage.
- Do not add backend services, auth, cloud sync, AI tutor chat, analytics services, rewards, streaks, or new game modules.
- Do not deploy unmerged application code to production in this PR.
- Do not add GitHub workflow automation, secrets, or committed `.vercel` project files.
