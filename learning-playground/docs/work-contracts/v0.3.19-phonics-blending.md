# v0.3.19 Word game — sound blending (c-a-t → cat)

Fourth slice of the Word-game build arc (issue #27). The next rung up the
pre-reading ladder: the child sees a word broken into sounds and taps the whole
word. Reuses the `phonics-match` runtime and Pip; adds the `blending` skill.

## Before Code

### Root Cause

The Word game teaches initial sounds (which sound a word *starts* with) but never
blending (putting sounds together to read a whole word) — the next rung. There is
no `blending` skill in the curriculum, no blending activity, and the runtime has
no way to show a word's sounds broken out. Without rhyming distractors a "blend"
task collapses back into initial-sound matching, so the discrimination must force
attention to every sound.

### Correct Fix Must Touch

- `src/content/curriculum/curriculum.v1.json` — add the `blending` skill (phonics
  domain, prerequisite `letter_sound_match`, contiguous levels, evidence/review
  like the other phonics skills, planned transfer contexts covering the
  activities' `context_type`). Add `blending` to `letter_sound_match.unlocks`
  (graph symmetry; `unlocks` has no consumer outside curriculum-graph, so no
  behavior change).
- `public/assets/images/{bat,hat}.svg` (new) — two flat clip-art cards for the
  `-at` rhyming set; `cat.svg` is reused. Render-verified.
- `src/content/activities/blend-{cat,hat,bat}.json` (new) — segmented-prompt
  blends over the same `{cat,bat,hat}` picture set (rhyming distractors force
  full-word blending). `interaction_model: tap_to_match`, `domain: phonics`,
  `skill_ids: [blending]`, `content.segments` = the sounds. Chained
  blend-cat → blend-hat → blend-bat (blend-bat terminal).
- `src/content/activities/phonics-find-t.json` — add `next_activity_id`
  (`blend-cat`) so the initial-sound chain climbs into blending (the child taps
  "Next", not auto-routed).
- `src/content/activity-catalog.ts` — register the three blend activities after
  the initial-sound block (they target `blending`, so they do not affect the
  initial_sound/letter_sound_match transfer recommendation regardless of order).
- `src/modules/phonics-match/PhonicsMatchActivity.ts` — read `content.segments`;
  when present, render a "sound-out" strip (one chip per sound) and rest Pip's
  mouth on the first sound. Whole-word correct tap → Pip cheers + picture pops
  (unchanged). No dispatch change (blends are tap_to_match/phonics).
- `src/styles/child-ui.css` — `.phonics-soundout` strip styles.
- `tests/modules/phonics-match.test.ts` — a blend renders its sound-out strip and
  Pip rests on the first sound; a correct whole-word tap completes + cheers; the
  full Word-game chain now runs find-b → … → find-t → blend-cat → blend-hat →
  blend-bat and terminates (updates the slice-2 chain test to the longer chain).
- `package.json` / `package-lock.json` — version bump to `0.3.19`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- The `initial_sound` / `letter_sound_match` skills' existing levels, evidence,
  and transfer contexts — untouched (only `letter_sound_match.unlocks` gains
  `blending`).
- The parent transfer recommendation for initial_sound (banana / reverse_mapping)
  — blend activities target a different skill and are registered after the
  initial-sound block.
- The `ActivityAttemptEvent` shape/emission, grid lock, hint, correct-choice
  resolution, Pip's cheer/alive behavior — unchanged.
- The activity JSON **schema** — `content.segments` rides the opaque `content`
  object; no schema change.
- `TapChoiceActivity.ts` (Math/Shapes) — untouched.
- **No reward loop, no auto-difficulty routing** — the chain is child-tapped and
  parent-approved; every blend keeps `requires_parent_approval: true`,
  `external_links_allowed: false`, no URLs.
