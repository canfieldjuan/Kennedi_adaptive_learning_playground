# v0.3.12 Illustrated Bear (art-standard proof)

First slice of the **illustrated art standard** (issue #3) — Juan chose illustrated
over emoji. This is a **style proof for reaction**, scoped to the Bear Cafe's bear
character. Do not merge on style approval from Juan (the look is his call), not on
green-checks alone.

## Before Code

### Root Cause

The Bear Cafe characters are system emoji glyphs (`content.character.icon`,
🐻/🐻‍❄️) with an emoji reaction accent (💭/✨/💛). Issue #3's "decide + apply an art
standard" is resolved to illustrated; nothing illustrated exists for the bear yet.

### Correct Fix Must Touch

- `src/modules/kennedis-orders/bear-art.ts` (new) — a local inline-SVG bear with
  three expressions (waiting / receiving / happy), tinted per caller.
- `src/modules/kennedis-orders/KennedisOrdersActivity.ts` — render the SVG bear at
  the five bear sites (phone portrait, order card, delivery window, handoff,
  complete), using each site's reaction; retire the now-redundant emoji reaction
  accent (the SVG face expresses the reaction).
- `src/styles/child-ui.css` — a `.bear-art` sizing rule (scales with each
  container's font-size, matching the emoji footprint).
- `package.json` / `package-lock.json` — version bump to `0.3.12`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- **Safety boundary:** SVG is authored inline (no external assets, no network, no
  deps); no `http(s)` introduced.
- No change to activity JSON / `content.character` (the emoji `icon` stays in the
  data; the runtime just renders the SVG instead) — so no schema change and the
  data is untouched if we revert.
- No change to the evidence model, event flow, mastery, transfer, storage, parent
  panel, home grid, or the child gate.
- Preschool-safe: friendly, rounded, calm.

### Intentional / to reconcile

- Retires the v0.3.5 emoji reaction accents at these sites (SVG expression
  supersedes them). Leaves the now-unused `.bear-cafe-reaction*` CSS in place for
  this proof; it is dead and will be removed in the follow-up if the look is
  approved.
- Only Kennedi's Orders is touched. Nature Camera Safari (not on main) and the
  other emoji surfaces (foods, scene) are deliberately out of scope until the
  style is approved — this is a proof, not the full rollout.

## Cold Diff Audit

### Change By Change Reconstruction

- `bear-art.ts` (new) — `renderBearArt(characterId, face)` returns an inline SVG:
  ears + head + tinted muzzle/cheeks/nose from a per-caller palette
  (`baby-polar-bear` white, `daddy-bear`/`mama-bear` brown), with a
  `faceFeatures()` expression (waiting: calm eyes + soft smile; receiving: wide
  bright eyes + open mouth; happy: upturned eyes + big smile).
- `KennedisOrdersActivity.ts` — imports `renderBearArt`; the five bear sites now
  render it with the site reaction; the `BearReaction`/`BEAR_REACTION_GLYPH`/
  `bearReactionAccent` emoji-accent block is removed.
- `child-ui.css` — adds `.bear-art { width/height: 1.25em; display: block }`.
- version → `0.3.12`; `MVP_BASELINE.md` note.

### Verification

- `npm run typecheck` — passed.
- `npm test` — 31 files / 171 tests passed (no test rendered the emoji accents;
  the pure engine tests are unaffected).
- `npm run build` — passed.
- Rendered all 9 variants (3 expressions × 3 caller tints) in-browser without
  errors. Automated **screenshots are unavailable** in this environment (the
  capture call times out), so the **visual/style judgement is Juan's**, via the
  Vercel preview and a rendered grid — that is the whole point of this proof.
