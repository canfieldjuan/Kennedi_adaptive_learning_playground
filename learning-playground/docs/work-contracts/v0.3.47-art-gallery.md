# v0.3.47 Bear Art Studio slice 3 — the mini gallery

Third art slice: finished pieces become a collection. The spec's completion
fantasy ("art card slides into mini gallery") built the local-first way: the
gallery is a **pure view derived from the event log** — no new storage key,
no export/clear changes, nothing to persist that isn't already persisted.

## Before Code

### Root Cause (slice)

Finished art vanishes. The gallery beat (slice 2) frames a piece for a
moment, but the child has no collection — nothing that says "these are the
pictures I made", no pride surface, no reason to look back. The spec listed
"simple gallery/finished card" as a core studio element and excluded only a
*complex save/export gallery*.

### Required Change Surface

- `src/core/art-gallery.ts` (new, DOM-free) — `GalleryPiece` +
  `deriveArtGallery(events)`: filter `completed` events with
  `metadata.game === 'bear-art-studio'`, map each mode's completion
  metadata to a small descriptor (mode, surface, sticker ids, color ids,
  pattern), most recent first, **capped at 12** (no unbounded growth);
  malformed metadata skipped fail-safe. Also
  `derivePieceFromCompletion(...)` shared with the runtime so the live
  piece and the derived history can never disagree. Persistence, parent
  export, and Clear Progress Data all come from the event log — this
  module stores nothing.
- `src/modules/bear-art-studio/gallery-art.ts` (new) — `galleryMiniSvg`
  renders a piece descriptor as a tiny framed card (stickers / painted
  subject / pattern stripes / shirt), closed studio palette map, ink
  frame. Render-checked.
- `src/modules/bear-art-studio/BearArtStudioActivity.ts` —
  - options gain `gallery?: GalleryPiece[]` (derived, injected by the app
    shell like `parentGuidance` elsewhere);
  - `free_decorate` completion metadata enriched with `sticker_ids` +
    `card_color_id` (the other modes' completion metadata already carries
    their descriptors);
  - on finish, every mode shows the **gallery shelf** in the complete
    area: the just-made piece slides in first (one-shot, reduced-motion
    guarded), followed by up to three earlier pieces.
- `src/app/main.ts` — pass `gallery: deriveArtGallery(getEvents())` into
  the studio render call. No other routing change.
- `src/styles/child-ui.css` — shelf strip + mini cards, slide-in
  keyframe + reduced-motion entry, phone sizes.
- Tests — `tests/core/art-gallery.test.ts` (derivation: filtering, order,
  cap, per-mode mapping, malformed-metadata safety);
  `tests/modules/bear-art-studio.test.ts` (shelf renders on finish with
  the live piece + injected history; enriched free-decorate metadata;
  no shelf before completion). Viewport suite run locally (CI gate).
- `docs/captures/bear-art-studio/` — gallery shelf desktop + mobile.
- `package.json` / `package-lock.json` — `0.3.47`; `MVP_BASELINE.md` note.

### Explicit Non-Scope / Must Not Change

- **No new storage key, no export schema change, no clear-button change**
  — the gallery is derived; the event log already owns persistence.
- No gallery management UI (no child-facing delete/share/rename), no
  full-screen gallery view, no home-screen changes, no screenshots/image
  files — minis re-render from descriptors.
- Game rules, evidence semantics, engines, schema, parent lane, other
  games — untouched (the only event change is additive completion
  metadata).
- Cap stays 12; the shelf shows at most 4.

### Assumptions / Blockers

- Completion metadata is the single source of truth for a piece's look;
  enriching free-decorate's completion event is additive (metadata is an
  open record; no consumer pins its exact key set — verified).
- Codex's five-flowers variant reuses the quantity mode, so its finishes
  join the gallery automatically.
- Blockers: none known.

## Contract Amendments

- **Finish now scrolls the complete area into view.** At 1366×900 the
  shelf + Next/Home landed below the fold on the color-request screen —
  the child could finish and see no way forward. `finishPiece` does an
  instant (non-animated) `scrollIntoView`, so reduced-motion needs no
  special case. Caught by looking.
- **Warm-scene feedback pill fixed (latent, all warm games).** The phone
  media block gives `.activity-feedback` a dark navy pill designed for
  white text; every warm scene (studio, workshop, station, shape garden)
  overrides the text to ink, producing dark-on-dark — first surfaced by
  this slice's mobile capture, present since the visual arc. Scoped
  additive fix: the four warm containers pair ink text with a cream pill;
  the dark pill remains for dark-background games. Caught by looking.

- **Shelf `[hidden]` display fix (Codex review catch, confirmed).** The
  shelf's `display: flex` author rule overrode the UA's
  `[hidden] { display: none }`, so the empty bordered strip was visible
  during play — the unit test asserted the `hidden` property (true) and
  the capture pass only covered completion states, so both missed it.
  Fixed with an explicit `[hidden]` rule, pinned by a CSS contract
  assertion, and verified live (computed display `none` during play,
  `flex` after finish).
