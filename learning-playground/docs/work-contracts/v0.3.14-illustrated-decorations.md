# v0.3.14 Illustrated Decorations

Third surface of the illustrated art standard (issue #3), after the approved bear
and foods. Completes the Bear Cafe item art: the four decorate-step toppings.

## Before Code

### Root Cause

After the illustrated bear (#22) and foods (#23), the decorations (bubbles 🫧,
hearts 💗, sprinkles ✨, stars ⭐) are the last emoji in the Bear Cafe.

### Correct Fix Must Touch

- `src/modules/kennedis-orders/decoration-art.ts` (new) — `renderDecorationArt(id)`
  returning local inline SVG for the four decorations (unknown → star).
- `src/modules/kennedis-orders/KennedisOrdersActivity.ts` — render the SVG at the
  two decoration sites: the tray plate badge and the decoration choice buttons.
- `package.json` / `package-lock.json` — version bump to `0.3.14`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change

- Inline SVG only (no external assets/network/deps); no `http(s)`. Reuses the
  existing `.food-art` sizing class, so no new CSS.
- Activity JSON / `content.decorations` untouched (emoji `icon` stays in data;
  no schema change, revertible).
- No change to the evidence model, tray logic, mastery, transfer, storage,
  parent panel, home grid, or the child gate.
