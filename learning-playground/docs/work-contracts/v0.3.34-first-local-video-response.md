# v0.3.34 First Local Video and Vocabulary Response

## Before Code

### Code Read Cold

- The Video Vault manifest is valid but empty, so the runtime always follows its
  existing empty-state branch and no child can inspect a real approved clip.
- The v0.3.33 validator enforces repo-bundled local paths, MIME and extension
  agreement, bounded duration, parent approval, exposure-only evidence, and no
  autoplay or loop behavior.
- A video ending emits a global and per-skill `completed` event with
  `exposure_only` metadata. Existing evidence code records completion history
  but excludes it from counted attempts, accuracy, transfer success, and
  mastery.
- The activity catalog has no response activity associated with Video Vault.
  The runtime has no child-controlled route from an ended clip to a separately
  scored response.
- The generic tap-choice runtime already emits correct and incorrect events with
  selected answer, correct answer, attempts, response time, hint state, and the
  activity's transfer metadata. A vocabulary response does not need a new
  runtime or event shape.

### Root Cause

The intake and evidence boundary is now explicit, but there is no approved media
or paired child response. Consequently, Video Vault can neither deliver its
first real local experience nor collect honest vocabulary evidence. Treating
video completion as the answer would violate the evidence contract; adding a
standalone response with no explicit handoff would make the prompt ambiguous.

### Correct Fix Must Touch

- Add one short, finite, repo-created WebM clip and local poster for the scoped
  concept “Bear bakes bread.” The clip must communicate the action visually and
  with local narration, require manual playback, and contain no remote content.
- Add local image choices for bread and its distractors so the response does not
  require reading.
- Extend Video Vault item metadata with a required response activity id for real
  items. Validate its id format alongside the existing media boundary.
- Add one normal `tap_to_match` vocabulary activity asking what Bear baked. It
  must use a medium `different_prompt_mode` context, cite an originating brief,
  use three large visual choices, and retain gentle hint/feedback rules.
- Register the response activity in the existing catalog. Use the existing hash
  route and tap-choice runtime; no app-router or shared schema change is needed.
- After the video ends, reveal one large manual response action. It may navigate
  only when the child taps it; it must never auto-advance.
- Keep the video completion event exposure-only. The separate activity alone
  emits correct/incorrect vocabulary evidence.
- Add tests proving the real media and poster exist, the manifest and response
  activity agree, invalid response ids fail closed, the response is approved and
  local, no navigation occurs merely because playback ended, and the response
  produces counted evidence through the existing activity model.
- Advance package, baseline, and roadmap to v0.3.34 and record the remaining
  breadth risk honestly.

### Must Not Change

  `skill-outcomes.ts`, `progress.ts`, `adaptive-engine.ts`, content-gap rules,

- Shared activity/event/storage schemas, curriculum levels, prerequisites,
  transfer-strength mapping, and vocabulary thresholds.
- Parent approval, parent gate, difficulty application, export/reset,

- Home screen choices, route parsing, automatic child routing, and the absence
  of Video Vault from the four-tile home screen.
- Number Train, Bear Cafe, Words, Art, spatial activities, Coloring, Nature
  Camera Safari, or their behavior.
- No backend, auth, cloud sync, uploads, parent-import flow, IndexedDB migration,
  external media, external links, new dependency, autoplay, autoplay chaining,
  loop, reward, streak, timer, child-facing AI, or pressure language.
- The unrelated untracked `.github/workflows/` directory.

### Acceptance Standard

- The real manifest contains exactly one approved local WebM item no longer than
  300 seconds, with a local poster and a valid response activity id.
- Every referenced real asset exists under `public` and the clip contains both a
  video and an audio stream.
- Video playback remains manually started and does not loop or auto-play another
  clip or activity.
- Ending the clip emits only exposure completion and reveals, but does not
  trigger, the response action.
- Tapping the response action opens a registered, parent-approved tap-choice
  activity with three visual choices and no required reading.
- Correct/incorrect vocabulary evidence comes only from the response activity;
  video completion still creates zero counted attempts and no mastery promotion.
- Existing activity, curriculum, safety, parent-approval, mobile-child-UI, and
  v0.3.33 evidence-boundary tests remain green.
- Tests, typecheck, build, lint if present, `git diff --check`, media probing, and
  mobile browser QA pass.
