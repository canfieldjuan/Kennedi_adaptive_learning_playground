# v0.3.32 Number Train — child UI polish + extension seams (arc slice 6)

Final slice of the Number Train build arc (issue #43): mobile-viewport polish,
accessibility tightening, cleanup verification, and the documented extension
seams for the future Math world. No new mechanics.

## Before Code

### Root Cause (slice)

The game is functionally complete but was built desktop-first: the phone
layout only shrinks the engine and seats (no journey/path/load-control
compaction, nothing for the phone-landscape breakpoint every other child
surface handles); the journey strip announces a static label rather than the
current station; timer cleanup on Home navigation is implemented but not
test-proven; and the arc-mandated extension-seams documentation (§13 of the
build spec) does not exist yet.

### Correct Fix Must Touch

- `src/styles/child-ui.css` —
  - extend the number-train `≤768px` rules: compact journey gap and station
    markers, smaller path stops and numeral type, tighter load controls;
  - add number-train rules to the existing phone-landscape breakpoint
    (`≤940px × ≤480px landscape`): compact engine/seats/path/journey so a
    round fits the short viewport without shrinking the tappable numeral
    choices or load controls below child-safe sizes (they stay on the shared
    `activity-choice` / `child-button` sizing the mobile contract governs).
- `src/modules/number-train/NumberTrainActivity.ts` — the journey strip's
  accessible label updates per round ("Trip: station X of Y"); no other
  behavior changes (arrival intentionally reuses the correct-answer cue — no
  new sounds, no celebration escalation).
- `tests/modules/number-train.test.ts` — journey label updates as the trip
  advances; destroy clears pending timeouts (spy on `window.clearTimeout`
  after a wrong-tap schedules the bounce reset).
- `tests/contract/mobile-child-ui.test.ts` — Number Train joins the mobile
  contract assertions (compact seat/journey rules exist at both breakpoints).
- Extension seams documented below (spec §13) — descriptions, not systems.
- `package.json` / `package-lock.json` — version bump to `0.3.32` (renumbered after Codex took 0.3.31 on main via #50).
- Docs note in `MVP_BASELINE.md`.

### Extension seams (documented, deliberately not built)

- **Math world menu** — the Math tile routes to `#activity/number-train`
  (`HomeScreen.ts`); a future Math world screen replaces that one route
  string and lists games. Number Train needs no change: it is one
  game-keyed envelope (`content.game: "number-train"`) dispatched like Bear
  Cafe, so sibling games register the same way with their own game ids.
- **Number Trail / Balance Garden** — new games get their own modules and
  game-keyed envelopes; they must NOT extend `NumberTrainActivity` or its
  types (deliberately narrow). Shareable domain-level pieces if and when
  needed: the structured-quantity visual (`buildTrainVisual`'s 2×5
  cars-of-ten model), numeral-choice rendering, and the number-path
  rendering — extract on second use, not before.
- **Range 21–50** — the data model and validator already accept 0–50
  (`NUMBER_TRAIN_ABSOLUTE_MAX`); `buildTrainVisual` renders up to five cars
  today. The seam is authored session configs in the envelope's opaque
  content (`max_quantity`, `round_count`, `seed`, and future difficulty
  presets) — which also supplies the higher-difficulty producers the new
  skills' upper bands need (tracked with #35).
- **Comparison / one-more / one-fewer rounds** — new members of the
  `NumberTrainRound` union with their own `roundKindAt` positions,
  `validateRound` rules, renderer, and `roundSkillIds`/`hintedSkillIdsFor`
  attribution (a comparison round would introduce a `quantity_comparison`
  skill through the same curriculum discipline as slice 5).
- **Parent guidance bounding** — `getSessionConfig` reads the envelope's
  opaque content; parent-approved difficulty overrides could bound
  `max_quantity`/`round_count` through the existing
  `parent-difficulty-application` machinery once a Number Train-aware
  override type is designed (deliberately not wedged into the tap-choice
  override this arc).

### Explicit Non-Scope / Must Not Change

- No new round types, mechanics, sounds, or celebration changes; no Math
  world screen, no Number Trail/Balance Garden scaffolding, no generic game
  engine.
- Round generation, evidence semantics, curriculum — untouched (slice 5 is
  the last semantic change of the arc).
- Tap-target sizes for interactive controls never shrink below the shared
  child-safe classes; only decorative/display elements compact.
- Protected modules (`tap-choice/`, phonics, Bear Cafe, `src/core/`,
  schema, curriculum) — untouched. `CURRENT_STATUS_AND_ROADMAP.md` is left
  to the Codex doc-alignment cadence that has been maintaining it.
