# v0.3.44 Counting Band Coverage

Fill the first catalog-reported difficulty gap with one bounded variant in an
existing game.

## Before Code

- The counting ladder defines level 1 as "Counts structured quantities
  accurately" over difficulty 2-3 at
  `src/content/curriculum/curriculum.v1.json:301-306`.
- The approved counting catalog currently jumps from difficulty-1 activities
  to Bear Cafe activities at difficulty 4 and 5. The exact missing rung is
  pinned at `tests/core/difficulty-coverage.test.ts:72-85`.
- Bear Art Studio already has a counting-only `quantity_decorate` activity at
  `src/content/activities/art-studio-three-stars.json:1-35`. It is difficulty
  1, asks for three stars in six spots, and currently chains directly to the

- The existing runtime creates one large tappable spot per authored slot,
  permits add/remove play without scoring, and judges only the explicit Check
  action at `src/modules/bear-art-studio/BearArtStudioActivity.ts:676-747`.

  counts through eight at
  `src/modules/bear-art-studio/BearArtStudioActivity.ts:1141-1157`.
- Catalog registration is the source of approved playable activities at
  `src/content/activity-catalog.ts:41-87`; the counting curriculum context list
  currently names the baseline math, Number Train, and three-star studio
  contexts at `src/content/curriculum/curriculum.v1.json:320-324`.

### Root Cause

The curriculum and promotion system require counting evidence from difficulty
2-3 for the structured-quantity rung, but the approved catalog contains no
counting activity in that band. The current game runtimes can represent the
skill, but no approved, reachable content configuration asks the child to
construct a larger structured quantity at the required difficulty. This is a
content-fit gap in the app, not a child performance problem.

### Correct Fix Must Touch

- Add one `art-studio-five-flowers` activity using the existing Bear Art Studio
  `quantity_decorate` mode.
- Author exactly five flowers over six fixed, child-safe spots at difficulty
  3. The extra spot must preserve an over-count path; Check remains the only
  scoring action.
- Declare counting as the only skill and `same_format_new_examples` as the
  transfer context. This activity improves difficulty coverage but must not be
  mislabeled as stronger transfer.
- Insert the new variant between the existing three-star and pattern activities
  so it is reachable in the current Art chain without a route or runtime change.
- Register the activity in the approved catalog and counting curriculum context
  list.
- Update focused tests to prove the new activity:
  - is valid, local, parent-approved content;
  - renders six large spots through the existing runtime;
  - records under-count, exact-five completion, counting-only skill IDs, and
    difficulty 3 through the existing event shape;
  - makes counting level 1 covered while preserving blocked-state evaluator
    coverage with an intentionally filtered fixture;
  - removes only the counting level-1 entry from the pinned gap inventory.
- Advance package, baseline, roadmap, and verification state to v0.3.44.
