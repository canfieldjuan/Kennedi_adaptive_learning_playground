# v0.3.9 Idle Nudge

Sixth slice of issue #3 (game richness backlog) applied to Kennedi's Orders — the
last self-contained item before the (operator-owned) art-direction decision.

## Before Code

### Root Cause

Nothing prompts the child back to the task while they pause on the make/fix
stage (issue #3 "No idle nudge / ambient motion"). Continuous ambient motion was
already added by the reaction accents (v0.3.5, the bobbing waiting cue); what
remained missing is a gentle re-engagement nudge after a genuine pause.

### Correct Fix Must Touch

- `src/modules/kennedis-orders/KennedisOrdersActivity.ts` — after a period of no
  interaction on the make/fix stage, gently nudge the order card; reset on any
  interaction; clear on teardown.
- `src/styles/child-ui.css` — a `.bear-cafe-order--nudge` pulse + `cafeIdleNudge`
  keyframe, added to the `prefers-reduced-motion` guard.
- `package.json` / `package-lock.json` — version bump to `0.3.9`.
- Docs note in `MVP_BASELINE.md`.

### Must Not Change (safety)

- **Not a nag / autoplay loop.** The nudge is a single bounded 3-pulse visual
  cue on the order card, fires once per idle period, and re-arms only on the next
  interaction. No speech is triggered (no autoplay chain), and it does not loop.
- The idle timer is cleared at the top of every render (so any interaction, or a
  stage change, resets it) and again on teardown — an interrupted or navigated
  view cannot leave a live timer or fire after unmount (the timer path only
  toggles a CSS class; it emits no events).
- No change to the evidence model, activity JSON, schema, mastery, transfer,
  recommendation, storage, parent panel, home grid, or the child gate.
- Motion stays preschool-safe: calm scale/glow pulse, bounded, reduced-motion-
  guarded.
