# v0.3.49 Bear Art Studio slice 5 — the canvas pass

The interaction pass: from "choose icon → click empty box" to "choose
tool/color → draw, fill, decorate a canvas". Same game, same evidence
system; the art table finally feels like an art table.

## Before Code

### Root Cause (slice)

Bear Art Studio plays like slot placement: the child picks an icon and
clicks a blank box. That is mechanical and does not match the drawing/
coloring/decorating fantasy. The screen also spends space on a large title,
a prompt paragraph, and text buttons that a non-reading child cannot use —
noise that competes with the art surface.

### Required Change Surface

- `src/modules/bear-art-studio/art-canvas.ts` (new) — the drawing surface:
  a `<canvas>` brush layer + free-placement sticker layer inside one easel
  element, driven by pointer events (mouse + touch). Forgiving round-cap
  strokes, no precision requirement; coordinates scaled to a fixed internal
  resolution. Test-safe: all 2D-context calls are guarded so behavior tests
  (mock DOM, no canvas context) exercise the evidence paths.
- `BearArtStudioActivity.ts` — three modes move to the canvas:
  - **free_decorate** (all four surfaces): brush (draw anywhere), bucket
    (repaint the surface backdrop — shirt/poster/wall/card), stickers
    (place anywhere; NO grid slots). Done button (large check icon)
    enables after the first canvas action.
  - **color_request**: brush + bucket. The bucket target is the existing
    subject region; a finished brush stroke is also an attempt. Using the
    requested color (either tool) completes; a non-target attempt is the
    existing gentle incorrect/hint flow.
  - **quantity_decorate**: free sticker placement on the canvas (tap a
    placed sticker to remove it); completes when the requested count is
    reached (spec supersedes the Check-button: placement IS the count).
    Requested/placed metadata preserved; per-placement events bounded by
    the target count.
  - pattern / story / fix keep their guided interactions (the spec allows
    guided slots outside free mode; fix already IS the bucket pattern).
  - Tool tray replaces the swatch/sticker rows: brush / bucket / sticker
    tool chips + swatches + sticker chips (sticker chips shown when the
    sticker tool is active). Selected tool + color highlighted.
- **Noise reduction (all studio modes)**: Home/Repeat become icon-only in
  the studio at every width (aria-labels keep the names); the title and
  prompt paragraph become screen-reader-only (the spoken prompt + visual
  request card carry the ask); the canvas/board becomes the dominant
  element.
- Brush evidence (summarized, never per pointer-move): `first_brush_mark`
  once, one `color_used` per distinct color (bounded by palette size),
  and the completion summary (`colors_used`, `canvas_action_count`,
  `placed_sticker_count`, `tool_mode` metadata).
- Activity JSONs — version bumps (interaction changed) for
  free-decorate, dress-bear, stage-poster, wall-picture, pink-request,
  three-stars; `slot_count` becomes inert for canvas modes (kept in
  content for compatibility; Codex's five-flowers keeps working
  unchanged).
- `tests/contract/validate-activities.test.ts` — the quantity
  `different_interaction_model` detector drops its slot-grid clause
  (free placement is the interaction now); still requires a positive
  integer target.
- `docs/contracts/art-studio.contract.md` — the loop/mode table updated
  to the tool model.
- CSS — easel-dominant layout (compact top strip / large canvas / bottom
  tray / compact feedback), tool chips, placed-sticker layer, done check,
  phone rules. Slot styles remain for pattern/story/fix.
- Tests — rewritten free/quantity behavioral tests (canvas actions via
  dispatched pointer/click events; mock gains `getBoundingClientRect` +
  guarded-context awareness), new brush-evidence tests (no per-move
  events), color-request brush path, existing story/pattern/fix suites
  stay green.
- `docs/captures/bear-art-studio/` — canvas free-art (drawn strokes +
  stickers), quantity, mobile; WebP.
- `package.json` / `package-lock.json` — `0.3.49`; `MVP_BASELINE.md`.

### Explicit Non-Scope / Must Not Change

- No new game, no flood-fill algorithm (bucket = large region/backdrop
  repaint per spec), no gallery changes (completion metadata keeps the
  keys the derivation reads), no engine/schema/parent/mastery/transfer
  changes, no dependencies, no camera/microphone/AI/remote anything.
- Pattern, story, and fix interactions unchanged this slice.
- The studio scene (visual arc) stays — it is inert background, not a
  panel.
- Event volume stays bounded: never an event per pointer move; sticker
  placement events only where a goal makes them meaningful.

### Assumptions / Blockers

- The existing gallery derivation reads `sticker_ids`/`card_color_id`/
  `applied_quantity`-style completion metadata — the canvas completions
  keep those keys populated so minis keep rendering.
- Auto-complete-at-N for quantity is a deliberate evidence-semantics
  change ordered by the spec ("game completes when the requested number
  is placed"); over-count evidence disappears (placement can never
  exceed N; removal exists below N). Documented here, detector updated.
- Blockers: none known.

### Verification Plan

- `npm run typecheck` / `npm test` / `npm run test:viewport` /
  `npm run build` / `git diff --check` (no lint script); live captures
  incl. real drawn strokes; expect a Codex thread at merge time.

## Contract Amendments

- **Done became a floating check chip on the easel** after the desktop
  capture showed "Finish art" below the 900px fold — a child could paint
  and never see how to finish. The chip floats on the easel corner
  (always visible; taps on it never reach the paint layer). Caught by
  looking.
- **The `[hidden]`-vs-author-display trap recurred** on the sticker row
  (`display: flex` defeating `hidden` when the brush tool is active) —
  same class of bug Codex caught on the gallery shelf. Fixed and pinned
  with a CSS contract assertion.
- **Dead card-variant CSS removed** (`__card--shirt/--poster/--wall-frame`
  and their slot-stacking union) — replaced by the easel variants; the
  guided modes keep the plain `__card` styles.
- **Codex's five-flowers test rewritten with the new semantics** (its
  quantity variant now free-places five flowers; same evidence keys).
