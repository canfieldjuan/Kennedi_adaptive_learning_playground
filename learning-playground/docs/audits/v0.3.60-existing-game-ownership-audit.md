# v0.3.60 Existing Primary-Game Ownership Audit

## Verdict

**No existing primary game currently satisfies the complete Ownership
Completion Contract.** Each game already has at least one useful foundation,
but all five fail one or more adoption-gate requirements. This report is an
audit, not retrofit authorization; current runtime behavior is unchanged.

The audited primary-game set is exactly Words / Word Builder, Bear Cafe,
Number Train, Bear Art Studio, and Story Stage. That set is declared by the
merged principle work contract; legacy direct-route activities and the absent
Nature Camera Safari are outside it
(`docs/work-contracts/v0.3.59-ownership-completion-principle.md:82-85`).

## Method And Classification

The audit traced the current route wiring, child action, payoff rendering,
durable state, export/reset boundary, and evidence emission for each game. It
then exercised representative completion paths in the running app at desktop
`1280x800` and mobile `390x844` viewports.

The classifications are:

- **confirmed**: current code meets the named boundary and the claim has a
  source citation;
- **contradicted**: current code proves the boundary is absent or violated;
- **could-not-determine**: current source and reachable browser behavior do not
  prove either result.

The governing acceptance boundaries are transformative action, an unscored
choice, one faithful completion object, exact continuity, child-controlled
inspection, child-facing bounded revisit, evidence separation, and preschool-
safe bounded flow (`docs/contracts/ownership-completion.contract.md:162-181`).

## Gaps Beyond The v0.3.59 Planning Snapshot

The earlier planning table correctly identified the missing Cafe order wall,
Words stage, Train route map, exact Art replay, and child Story shelf
(`docs/work-contracts/v0.3.59-ownership-completion-principle.md:113-139`). The
independent audit found three additional contract failures that change retrofit
priority:

1. **Bear Cafe preference is scored as knowledge.** Free Make has no required
   order and accepts any food (`src/modules/kennedis-orders/KennedisOrdersActivity.ts:933-943`),
   but the food-selection evaluator returns `correct`
   (`src/modules/kennedis-orders/KennedisOrdersActivity.ts:1010-1017`) and the
   event builder assigns that result to every activity skill
   (`src/modules/kennedis-orders/KennedisOrdersActivity.ts:1293-1309`). The
   shipped Free Make activity labels the preference as vocabulary evidence
   (`src/content/activities/kennedis-orders-free-make-001.json:2-17`).
2. **Art Studio preference creates skill evidence.** The shipped free-decorate
   activity assigns `color_fill` to open-ended colors and stickers
   (`src/content/activities/art-studio-free-decorate.json:2-16`), and every
   emitted event maps its outcome onto every activity skill
   (`src/modules/bear-art-studio/BearArtStudioActivity.ts:302-345`). A completed
   preference therefore becomes completion evidence for `color_fill`
   (`src/core/evidence.ts:72-74`, `src/core/evidence.ts:137-147`).
3. **Story history is not bounded.** Reads fail safely and strip unknown fields
   (`src/core/storage.ts:416-427`), but every session append grows the stored
   array without a cap (`src/core/storage.ts:431-438`). The ownership contract
   requires collections not to grow without limit
   (`docs/contracts/ownership-completion.contract.md:127-144`).

No required boundary remains **could-not-determine** after the source and
browser pass.

## Ordered Retrofit Sequence

The independent audit changes the old planning order because it found active
evidence-boundary and resource failures that the snapshot omitted.

1. **Bear Cafe** — first because an answerless preference currently emits
   `correct` vocabulary evidence, then because the delivered order disappears.
2. **Bear Art Studio** — next because exact authored work is discarded and an
   expressive completion currently becomes skill evidence.
3. **Story Stage** — cap the unbounded record and connect the already-strong
   completion object to a child shelf and exact replay.
4. **Words / Word Builder** — add embodied word effects, one unscored detail,
   and a preserved word stage without changing phonics evaluation.
5. **Number Train** — preserve an exact journey and one light train choice after
   math, then expose the capped route map.

Each item is one game-owned work contract, isolated worktree, and PR. No shared
completion framework should be generalized before at least two game-specific
objects prove a common shape. This audit authorizes none of those implementation
steps; the operator must order each slice separately.

## Later-Slice Verification Standard

Every retrofit must prove:

- the same completion object before payoff and after child revisit;
- exact visible choice continuity at desktop and mobile viewports;
- revisit without completing another lesson;
- cap, malformed-data, export, and parent-clear behavior;
- no preference-derived correct, incorrect, hint, mastery, transfer, or
  recommendation signal;
- unchanged assessed answers, event timing, curriculum, mastery, transfer
  strength, parent approval, routes, and safety unless the slice's written
  contract explicitly reaches one of those causes.
